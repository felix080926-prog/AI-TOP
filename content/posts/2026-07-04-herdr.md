**大家好，我是雷达君。**

如果你同时开了三个 Claude Code 窗口，一个在重构后端，一个在写前端组件，还有一个在跑测试。你想看看它们各自进展如何，只能一个一个窗口切过去——这个卡住了，那个还在转圈，另一个好像已经跑完了但你不知道。

你像个救火队员一样在多个终端之间来回跳跃，手忙脚乱地切换上下文。

更麻烦的是，你关掉笔记本回家，第二天打开一看——昨晚跑了一夜的 Agent 因为会话超时，全挂了。

**你的 AI 小弟们都在埋头干活，但你看不见它们谁在忙、谁在等、谁已经干完了。**

今天雷达君要安利的这个开源项目，就是来解决这个问题的。它是一个**终端原生的 Agent 多路复用器（multiplexer）** ——你在一个终端窗口里就能看到所有 AI Agent 的实时状态，谁卡住了、谁在工作、谁已经干完了，一目了然。

它就是——**Herdr**。

## 01 ｜核心逻辑：把 tmux 重建成 Agent 的“作战指挥室”

如果你用过 tmux，Herdr 的核心逻辑一句话就能说清楚：**tmux 是为终端用户设计的，Herdr 是为 AI Agent 设计的。**

tmux 能让你在一个终端里开多个窗口、分多个面板，会话关掉后进程还在后台跑。但它有一个致命的问题——**它不知道哪个面板里跑的是 AI Agent，更不知道这个 Agent 现在是卡住了、在工作、还是已经干完了**。

你可以用各种 hack 手段去“猜”Agent 的状态——监听输出里的特定字符串、解析日志、写一堆胶水脚本——但每个 Agent 都要单独配置，而且你仍然没有一个统一的视图来看整个“舰队”的状态。

Herdr 把 tmux 的“持久会话”和“多面板”能力继承下来，然后做了一件 tmux 做不到的事：**它知道每个面板里跑的是什么 Agent，并且能实时显示每个 Agent 的状态——blocked（卡住了）、working（工作中）、done（已完成）、idle（空闲中）**。

你不再需要逐个窗口去翻看。一个侧边栏，所有 Agent 的状态一目了然。

**一句话：tmux 给你的是“多个终端”，Herdr 给你的是“一个 AI Agent 指挥中心”。**

## 02 ｜硬核功能盘点

> 🔗 **GitHub 地址**：https://github.com/ogulcancelik/herdr

### 1. 一个终端，盯住所有 Agent

Herdr 最核心的能力：你打开一个终端窗口，跑 Herdr，然后在这个窗口里启动你所有的 AI Agent——Claude Code、Codex、OpenCode、Cursor Agent、Devin CLI、GitHub Copilot CLI……每个 Agent 都有自己的**真实终端面板**，不是模拟的，是真正的 PTY（伪终端）。

右侧的侧边栏会实时显示每个 Agent 的状态：

- **blocked（卡住了）** ：Agent 在等你输入或确认
- **working（工作中）** ：Agent 正在干活
- **done（已完成）** ：任务跑完了
- **idle（空闲中）** ：Agent 在待命

支持的 Agent 列表非常长：Claude Code、Codex、OpenCode、Grok CLI、Hermes Agent、Devin CLI、Cursor Agent、Kimi Code CLI、GitHub Copilot CLI……以及 Pi、Droid、AMP、Kilo Code CLI、QoderCLI、Kiro CLI 等。Gemini CLI 和 Cline 也在检测列表中。基本上你听说过的 AI 编程工具，Herdr 都能识别。

### 2. 关掉笔记本，Agent 继续跑

这是 Herdr 从 tmux 继承过来、但对 AI Agent 场景至关重要的能力。

你在 Herdr 里启动了一个需要跑一整夜的代码重构任务。下班了，你直接合上笔记本。第二天到公司，打开终端，重新连上 Herdr——**所有 Agent 还在原地，该跑的运行着，该卡住的等着你输入**。

Herdr 在后台跑着一个常驻服务，所有面板和 Agent 进程都活着。你可以从任何终端重新接入——包括通过 SSH 从你的手机接入。Agent 不会因为你的终端会话断开而挂掉。

### 3. 工作区、标签页、面板：按项目组织你的 Agent 们

Herdr 支持**工作区（workspaces）、标签页（tabs）和面板（panes）** 三层组织方式。

你可以按项目来组织：一个工作区放“项目 A”的所有 Agent，另一个工作区放“项目 B”的。每个工作区里可以开多个标签页，每个标签页里可以再分多个面板。鼠标点击、拖拽、拆分面板，全部原生支持。

**你不需要记住任何快捷键就能上手**——Herdr 是鼠标优先设计的。当然，如果你是从 tmux 过来的，`ctrl+b`这套快捷键也完全兼容。

### 4. Agent 也能“指挥”Herdr：让 AI 自己编排自己

这是 Herdr 最硬核、也最面向未来的能力。

Herdr 暴露了一个**本地 Socket API 和 CLI**，Agent 可以通过这个接口来操控 Herdr 本身。一个 Agent 可以创建新的工作区、拆分面板、生成子 Agent、读取其他 Agent 的输出、订阅状态变化。

这意味着什么？你的**主 Agent 可以自己决定什么时候需要启动一个子 Agent 来处理特定任务，然后在一个新的面板里把它拉起来，等它跑完了再读取结果**——整个过程不需要你手动干预。

Herdr 还提供了一个可复用的**Agent Skill**，一条命令就能安装到你的 Claude Code 或 Codex 里：

```bash
npx skills add ogulcancelik/herdr --skill herdr -g
```

装完之后，你的 AI Agent 自己就知道怎么用 Herdr 了。

### 5. 单二进制，10MB，跑在任何地方

Herdr 是一个**~10MB 的 Rust 二进制文件**，没有任何依赖。支持 Linux 和 macOS，Windows 也在 beta 中。没有 GUI、没有 Electron、没有 Mac-only 的限制、不需要注册账号、不上传任何遥测数据。

你可以在本地跑，也可以在 VPS 上跑，然后从任何地方 SSH 连上去。甚至支持`herdr --remote`模式，让你的本地终端作为远程 Herdr 服务器的客户端，连粘贴图片都能正常工作——这是纯`ssh + tmux`做不到的。

## 03 ｜横向对比

Herdr、tmux 和 GUI Agent 管理器，这三者代表了完全不同的设计哲学：

| 维度               | **Herdr**                           | **tmux / Zellij**   | **GUI Agent 管理器** |
| :----------------- | :---------------------------------- | :------------------ | :------------------- |
| **核心定位**       | Agent 专用多路复用器                | 通用终端多路复用器  | Agent 桌面管理应用   |
| **Agent 状态感知** | ✅ 原生支持（blocked/working/done） | ❌ 不知道谁是 Agent | ✅ 能看到状态        |
| **运行位置**       | 终端内部                            | 终端内部            | 独立桌面应用         |
| **持久会话**       | ✅ 关掉终端继续跑                   | ✅                  | ⚠️ 部分支持          |
| **远程 SSH 接入**  | ✅                                  | ✅                  | ❌                   |
| **真实终端**       | ✅ 真正的 PTY                       | ✅                  | ❌ 模拟终端          |
| **Agent 可编程**   | ✅ Socket API + Skill               | ⚠️ 需脚本 hack      | ⚠️ 有限              |
| **单二进制/轻量**  | ✅ ~10MB                            | ✅                  | ❌ 通常很重          |
| **跨平台**         | Linux/macOS/Windows(beta)           | Linux/macOS         | 通常 Mac-only        |

数据来源：项目 README 及官方对比页面

**简单说：tmux 给你持久面板但不知道 Agent 状态，GUI 管理器知道 Agent 状态但把你锁在它的应用里。Herdr 在终端里同时给了你“持久面板”和“Agent 状态感知”** 。

## 04 ｜典型场景和避坑

**适合谁用？**

- **同时跑多个 AI Agent（Claude Code、Codex、OpenCode 等）的开发者**：一个终端看所有 Agent 的状态，不用来回切窗口
- **需要长时间运行 Agent 任务（过夜重构、批量测试）的团队**：关掉笔记本 Agent 继续跑，第二天回来接着看
- **在远程服务器上跑 Agent、需要随时接入查看的开发者**：Herdr 跑在 VPS 上，从任何地方 SSH 连进去
- **想让 AI Agent 自己编排自己的高级用户**：Socket API + Skill 让 Agent 能操控 Herdr

**不推荐给谁？**

- **只用单个 Agent、不需要同时管理多个的开发者**：直接跑 Claude Code 就完了，不需要多一层
- **完全不用终端、只用 IDE 内置 AI 的开发者**：Herdr 是终端工具，不适合你
- **Windows 用户**：Windows 版还在 beta，稳定性可能不如 Linux/macOS

## 05 ｜快速上手指南（2 分钟）

**第 1 步：安装**

```bash
# macOS / Linux
curl -fsSL https://herdr.dev/install.sh | sh

# Windows (PowerShell)
powershell -ExecutionPolicy Bypass -c "irm https://herdr.dev/install.ps1 | iex"
```

也支持 Homebrew、mise、Nix 等包管理器。

**第 2 步：启动 Herdr**

```bash
herdr
```

你会看到一个空白的终端界面，右侧有一个状态侧边栏。

**第 3 步：启动你的 Agent**
在 Herdr 里创建一个新面板（鼠标右键或`ctrl+b %`），然后启动你的 Agent：

```bash
claude
# 或
codex
# 或
opencode
```

右侧侧边栏会自动识别这个 Agent，并显示它的实时状态。

**第 4 步：关掉终端，再重新连回来**

```bash
# 关掉终端窗口，或者直接退出
exit

# 重新打开终端，连回Herdr
herdr attach
```

所有 Agent 都还在原地，该跑的在跑，该卡住的等你输入。

> **验证安装成功**：运行`herdr --version`，如果返回版本号（如`herdr 0.4.0`），说明安装成功。

## 06 ｜总结

Herdr 解决了一个很朴素但很真实的问题：**你的 AI 小弟们越来越多了，但你看不见它们谁在忙、谁在等、谁已经干完了。**

你同时开着 Claude Code、Codex、OpenCode，每个都在处理不同的任务。你不知道哪个卡住了需要你介入，哪个已经跑完了在等你验收，哪个还在埋头苦干。你只能一个个窗口翻过去看——浪费的是你的注意力和时间。

Herdr 的答案是：**给 AI Agent 配一个“作战指挥室”。**

一个终端窗口，所有 Agent 的状态一目了然。关掉笔记本，Agent 继续跑。从任何地方 SSH 连进去，随时查看进度。甚至让 Agent 自己操控 Herdr，实现自我编排。

在 AI Agent 越来越普及的今天，我们需要的不只是“更聪明的 Agent”，还有“更好地管理 Agent”的工具。Herdr 就是那个工具。
