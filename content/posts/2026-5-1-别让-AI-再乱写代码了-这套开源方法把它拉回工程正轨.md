大家好，我是雷达君。

你有没有遇到过这样的情况：让 AI 帮你改个 bug，它直接重写整个文件。你说“加个功能”，它二话不说就开始堆代码。改完一跑，原有的测试全红了。

AI 很聪明，但它不知道什么叫**工程规范**。

它不知道写代码之前要先理解需求，不知道改代码之前要先写测试，更不知道什么叫“先设计、再计划、后执行”。

今天雷达君安利一个项目。它不是“更聪明的模型”，而是一套让 AI 乖乖按正规工程流程干活的方法论。支持 Claude Code、Cursor、Codex、Gemini CLI、GitHub Copilot 等主流 AI 编码工具。

它不是教 AI 怎么写代码，而是**告诉 AI 怎么按照正确的流程写代码**。

它就是——**Superpowers**。

## 01 ｜核心逻辑

说白了，你今天用的 AI 编码工具，比如 Claude Code，它很能干活。但问题是——你一开口说“我要做个功能”，它就直接跳进代码里开始写。需求没对齐、设计没确认、测试没写，全凭“直觉”往前冲。

Superpowers 做的事，就是在 AI 冲动之前，先**把流程拉回到稳当的工程节奏**。

你一说要做开发，它不急着写代码，先退一步反问：你到底想解决什么问题？

等从对话里抽出一份你说得清的 Spec，它分段给你确认——每段短到你愿意读，改到你满意为止。设计签字画押之后，它再帮你榨出一份实现计划，细致到**一个不熟悉项目的人也能按步骤完成**。

你点头说干，它才发射**子代理驱动开发**流程：一个任务一个任务往下啃，干完一个就自查、审代码，没有问题再往前走。

**工程规范不再是“好建议”，而是强制条。**

## 02 ｜硬核功能盘点

Superpowers 不是单一功能的插件，而是一套**完整的工程流程体系**。它把 AI 的工作拆成了四个强制阶段：

🔗 **GitHub 地址**：https://github.com/obra/superpowers

**阶段一：需求对齐（先别写代码）**

任何开发任务启动前，AI 会强制触发**需求梳理**。它不会直接写代码，而是先和你把问题聊透：“你到底想解决什么？”“现有的测试是否都通过？”生成一份简短的设计文档，逐段让你确认。不满意就改，改到你点头为止。

这一步相当于给 AI 装了个**产品经理过滤器**——需求没对齐，代码别想动。

**阶段二：任务拆解（2-5 分钟粒度）**

设计确认后，AI 会把整个开发任务拆成一个个“**2-5 分钟就能干完**”的小任务。每个任务都写清楚了：改哪个文件、写哪些代码、怎么验证。

AI 不用猜，照着干就行。你可以在关键节点介入，也可以放手让它跑。

**阶段三：执行与隔离（不污染主分支）**

Superpowers 会在任务开始前，用**Git Worktree**自动创建一个隔离的开发分支，跑一遍项目初始化，验证测试基线是否干净。所有改动都在独立工作区进行，主分支不受影响。

然后，它会为每个小任务派一个全新的**子代理**去执行。子代理干完后，会先自查是否符合设计文档，再检查代码质量。关键问题直接卡住不放行，不放低级 bug 混过去。

_现实摩擦提醒：在非常复杂的项目里，子代理之间偶尔会出现上下文理解偏差，仍需你在关键节点做决策。但相比纯对话式开发，稳定性提升非常明显。_

**阶段四：验证与审查（不写测试，不给写代码）**

Superpowers 强制走**测试驱动开发（TDD）** 的红-绿-重构循环：先写一个会失败的测试，看着它红，再写最少代码让它绿。测试没写好，业务代码就别想动。

每个任务完成后，还会自动触发**代码审查**，按严重等级汇报问题。关键问题直接卡住不放行。

**一句话总结：AI 不是即兴表演，而是一条流水线。每一步都有章可循，每一步都强制执行。**

## 03 ｜典型场景和避坑

**适合谁用？**

- **日常用 AI 辅助开发、但总觉得 AI“不够稳”的开发者**：Superpowers 能强制 AI 按规范流程走，减少“乱写”的概率。
- **想让 AI 承担更多自动化工作的团队或个人**：子代理并行执行，AI 可以连续工作数小时不偏离计划（当然，关键节点还是需要你看一眼）。
- **重视代码质量、希望用 TDD 和 Code Review 规范 AI 产出的人**：Superpowers 把这些工程实践固化成强制流程，不是“建议”，是“必须”。

**不推荐给谁？**

- **只想让 AI 快速写个原型、不在乎规范流程的人**：Superpowers 会增加前置环节，不如直接让 AI 开干来得快。
- **项目没有测试、不想写测试的人**：Superpowers 强制 TDD，没有测试用例根本走不下去。
- **讨厌流程、喜欢自由发挥的开发者**：Superpowers 的本质就是给 AI 套上流程约束，不喜欢被框住的人可能会觉得烦。

## 04 ｜ 5 分钟快速上手

Superpowers 支持多种平台，这里以最常用的 Claude Code 为例：

**方式一（官方插件市场，最推荐）**

```bash
/plugin install superpowers@claude-plugins-official
```

**方式二（Superpowers 独立市场）**

```bash
/plugin marketplace add obra/superpowers-marketplace
/plugin install superpowers@superpowers-marketplace
```

**其他平台快速参考**：

- **OpenAI Codex**：打开插件搜索界面，输入/plugins，搜索 Superpowers，点击 Install Plugin
- **Cursor**：在 Agent 聊天中输入/add-plugin superpowers
- **Gemini CLI**：`gemini extensions install https://github.com/obra/superpowers`
- **GitHub Copilot CLI**：`copilot plugin marketplace add obra/superpowers-marketplace`
- **OpenCode**：告诉 OpenCode 跟随安装指引即可

**验证安装成功**：安装后启动一个开发任务，如果 AI 先问“你想解决什么问题”、而不是直接写代码，就说明 Superpowers 生效了。

## 05 ｜总结

Superpowers 做的事很简单：**不是让 AI 更聪明，而是让它更守规矩。**

需求对齐、任务拆解、TDD、代码审查、子代理执行——这些本来属于人类工程师的流程，被它一条条强制落实。

**AI 编程的上限，不取决于模型有多强，而取决于流程有多稳。**
