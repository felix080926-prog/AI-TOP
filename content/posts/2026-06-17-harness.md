大家好，我是雷达君。

你知道 Claude Code 很强大，但你有没有想过一个问题：**它到底是怎么工作的？**

51 万行代码，1884 个文件。当你看着它在终端里“刷刷刷”地干活，它背后到底执行了什么工具、调用了什么 API、每一步花了多少钱，你完全不知道。

想研究一下 Agent 架构？面对 51 万行代码，正常人看一眼就放弃了。

想自己改点逻辑？光是找到对应的文件就够你翻半天。

这不是你的问题。这是闭源软件和庞大体量共同制造的“知识垄断”。你知道它能做，但你看不见它怎么做。

今天雷达君要安利的这个开源项目，由香港大学数据科学研究所（HKUDS）出品，用**1.1 万行 Python 代码**重新实现了 Claude Code 的核心架构。砍掉了所有企业级冗余，只保留最纯粹的**Agent Harness**骨架。

你可以把它当成一个**“教学标本”**——打开就能看到 Agent 的每一次工具调用、每一步决策逻辑，完全透明、完全可控。

它就是——**OpenHarness**。

## 01 ｜核心逻辑：模型负责思考，Harness 负责干活

大语言模型非常聪明，能理解你的问题，生成高质量的回答。但它有一个致命缺陷——**它没有手**。

它不能读写文件，不能执行命令，不能搜索网页，不能记住上一次对话。LLM 本身只是一个**大脑**。要让这个大脑真正“动手干活”，你需要一套完整的**身体系统**。

这套身体系统，在学术界和工业界有一个专门的名字——**Agent Harness**（智能体框架）。

OpenHarness 对 Harness 的定义非常精炼：**模型提供智能，Harness 提供手、眼睛、记忆和安全边界**。

更直白一点：模型决定“做什么”，Harness 负责“怎么做”——权限校验、工具执行、结果格式化、错误重试，全部由 Harness 搞定。

OpenHarness 把这套架构拆成了 10 个核心子系统，其中最核心的是**Agent Loop**（代理循环）：

1. 模型接收用户指令，决定调用哪个工具
2. Harness 执行权限检查 → 触发前置钩子 → 执行工具 → 触发后置钩子 → 返回结果
3. 模型看到结果，决定下一步做什么
4. 循环往复，直到任务完成

**在这个循环里，每一步都是透明、可观测、可干预的。**

## 02 ｜硬核功能盘点

> 🔗 **GitHub 地址**：https://github.com/HKUDS/OpenHarness

OpenHarness 的架构由 10 个核心子系统组成，每个都有明确的职责。我把它们聚合成五个最值得关注的模块：

**1. 43 个内置工具：手、眼、记忆全覆盖**

OpenHarness 内置了**43+工具**，覆盖了 Agent 干活需要的所有基础能力：

- **文件操作**：Read、Write、Edit、Glob、Grep —— 读写改查，样样齐全
- **Shell 执行**：Bash 命令，带完整的权限控制
- **搜索能力**：WebFetch、WebSearch、ToolSearch —— 网页抓取和搜索
- **Agent 协调**：Subagent Spawning & Delegation —— 生成和委派子 Agent
- **MCP 协议**：MCPTool、ListMcpResources —— 接入 MCP 生态
- **任务管理**：TaskCreate、TaskGet、TaskList、TaskStop —— 后台任务全生命周期管理

每个工具都带 Pydantic 输入验证（结构化、类型安全）和自描述 JSON Schema（模型自动理解工具用法），还集成了权限检查和 PreToolUse/PostToolUse 钩子支持。

**2. Skills 按需加载：知识库不占上下文**

OpenHarness 的技能系统（Skills System）很有意思——**技能是按需加载的知识，只有模型需要时才加载**。

预置的 Skills 包括：commit（结构化 Git 提交）、review（代码审查）、debug（系统诊断）、plan（编码前设计）、test（编写和运行测试）、simplify（代码重构）、pdf 处理、xlsx 操作等 40+个。而且兼容 Anthropic 官方的 skills 生态，把`.md`文件复制进去就能用。

**3. 上下文与记忆：从“金鱼脑”到“大象脑”**

OpenHarness 在记忆管理上做了三层设计：

- **CLAUDE.md 自动发现与注入**：项目根目录的说明文档自动加载
- **上下文压缩**：会话过长时自动压缩，防止“越聊越笨”
- **MEMORY.md 持久记忆**：跨会话记住关键信息
- **会话恢复与历史**：中断了随时接着聊

**4. 安全与治理：不让 AI“乱来”**

OpenHarness 内置了多层安全护栏：

- **多级权限模式**：不同操作不同权限等级
- **路径级与命令级规则**：精细控制 AI 能碰什么文件、能跑什么命令
- **PreToolUse/PostToolUse 钩子**：工具执行前后插入自定义逻辑
- **交互式审批对话框**：敏感操作需人工确认

**5. 模型自由：想用哪个用哪个**

OpenHarness 支持 Anthropic Claude、OpenAI、Google Gemini，以及任何兼容 OpenAI 接口的端点。你也可以用 Ollama 在本地免费跑。自带 Token 计数与成本追踪，每笔账都算得清清楚楚。

> **现实摩擦提醒**：OpenHarness 当前版本（v0.1.7）处于早期阶段，虽然核心架构已经跑通，但部分高级功能（如 ClawTeam 集成）仍在规划中。生产环境大规模使用前建议充分测试。

## 03 ｜ OpenHarness vs 同类项目

Claude Code、OpenClaw、OpenCode 都是 AI 编程 Agent 领域的明星项目，但 OpenHarness 的定位跟它们完全不同：

| 维度         | **OpenHarness**              | **Claude Code**        | **OpenClaw**         |
| :----------- | :--------------------------- | :--------------------- | :------------------- |
| **核心定位** | Agent Harness 教学与研究框架 | 生产级 AI 编程助手     | 万能连接器/协议层    |
| **代码量**   | **1.1 万行**                 | 51 万行                | 53 万行              |
| **设计目标** | 透明、可学习、可扩展         | 功能全面、开箱即用     | 多渠道接入、协议优先 |
| **适用人群** | 研究者、学习者、框架开发者   | 日常编程的重度用户     | 极客、多渠道部署用户 |
| **学习曲线** | 低（代码量少、结构清晰）     | 中（功能多但文档完善） | 高（配置复杂）       |

OpenHarness 的价值不在于“功能最多”，而在于**用最小的代码量把 Agent Harness 的核心架构讲清楚**。如果你想理解 Agent 内部是怎么工作的、想自己改一个 Harness、想研究工具调用和权限控制的实现细节——OpenHarness 是唯一的选择。

## 04 ｜典型场景和避坑

**适合谁用？**

- **想学习 AI Agent 架构但被 51 万行代码劝退的开发者**：OpenHarness 用 1.1 万行代码把核心骨架讲清楚了
- **想研究 Agent 工具调用、权限控制、记忆管理等机制的 AI 研究者**
- **想基于 Harness 构建自己的定制化 Agent 的框架开发者**
- **希望 Agent 架构完全透明、可审计的安全敏感团队**

**不推荐给谁？**

- **只想“开箱即用”写代码、不想研究架构的普通用户**：直接用 Claude Code 或 Cursor 更省事
- **需要企业级稳定性、遥测和 OAuth 等生产级功能的生产环境**：OpenHarness 砍掉了这些，更偏向研究和教学

## 05 ｜快速上手指南

OpenHarness 的安装和使用极其简单，一个命令就能启动。

**第 1 步：安装**

```bash
# 通过pip安装
pip install openharness
```

**第 2 步：配置 API Key**

```bash
export ANTHROPIC_API_KEY="sk-xxxx"  # 或其他提供商
```

**第 3 步：启动**

```bash
oh  # 启动OpenHarness CLI
```

启动后，你会看到一个简洁的终端界面。输入任何任务，OpenHarness 会展示完整的工具调用链——每一步调用什么工具、传了什么参数、返回了什么结果、花了多少 Token，全部实时可见。

## 06 ｜总结

OpenHarness 解决的核心问题是：**Agent 的“黑盒”困局。**

Claude Code 很强大，但 51 万行代码让它成了一个“你只能用、不能看”的黑盒。OpenHarness 用 1.1 万行 Python 代码，把 Agent Harness 的核心骨架完整呈现了出来。

模型提供智能，Harness 提供手、眼睛、记忆和安全边界——这个理念在 OpenHarness 里被实现得干净、清晰、可扩展。无论你是想学习 Agent 架构的学生、想研究 Harness 机制的研究者，还是想构建自己定制化 Agent 的开发者，OpenHarness 都提供了最合适的起点。

安装包和完整代码都在 GitHub 上：https://github.com/HKUDS/OpenHarness
