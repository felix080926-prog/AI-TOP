### 01 ｜核心逻辑

Shannon 把自己定位成一个"白盒 AI 渗透测试员"。翻译成人话就是：把项目源码交给它，它像个真正的黑客一样，找出漏洞并且真刀真枪地利用给你看。

它的工作流程大概是三件事：

1. **读源码** —— 把你的整个项目扫描一遍，找哪里可能有问题
2. **找攻击面** —— SQL 注入？XSS？越权？SSRF？它自己判断
3. **发起攻击** —— 直接在跑着的应用上动手，给你看"你看，这里真的能被打穿"

这和市面上那些安全扫描工具完全不是一个路子。大多数扫描器是"黑盒"的——从外面乱拳打死老师傅，报一堆误报让你自己去筛。Shannon 是"白盒"的——它真的能理解你的代码逻辑，所以找到的不是纸上谈兵的"潜在风险"，而是实实在在可以被利用的漏洞。

名字也挺有意思。**Shannon（香农）**——致敬信息论之父克劳德·香农。往深了想，安全本质上就是信息不对称的战斗，用香农的名字给一个安全工具命名，挺有味道。

### 02 ｜硬核功能盘点

🔗 GitHub 地址：https://github.com/KeygraphHQ/shannon

**功能一：源码驱动的漏洞发现**
不是随机扫描，而是读懂你的代码再找问题。它能理解路由、数据流、输入输出关系，所以误报率比传统扫描器低了一个量级。

**功能二：自动化真实利用**
这是最骚的地方。大多数安全工具只会告诉你"这里可能有风险"，Shannon 不跟你来这套——它在你的应用上执行真实的 payload，给你看被攻破的截图或者数据。**结果就是：你没法假装没看见。**

**功能三：浏览器自动化攻击**
它用 Playwright/Puppeteer 这类浏览器自动化工具模拟真实攻击场景，能跑 XSS、CSRF、点击劫持这些需要浏览器环境才能验证的漏洞。

**功能四：覆盖 OWASP Top 10**
SQL 注入、NoSQL 注入、XSS、SSRF、认证绕过、IDOR……基本上排得上号的 Web 漏洞类型它都覆盖。特别值得一提的是 SSRF（服务端请求伪造），这个在很多扫描器里是盲区，但 Shannon 的代码分析能力让它能找到隐蔽的 SSRF 入口。

**功能五：提供 PoC（Proof of Concept）**
它不是只告诉你"有洞"，而是把完整的利用过程记录下来。你可以直接拿着 PoC 截图去跟团队说"修这个，不然会出事"——比口说无凭强一万倍。

**我的评价：** 这项目最聪明的地方是它选了一个"低垂果实"——安全扫描这件事已经被做了无数遍，但用 LLM 做白盒渗透，之前没人真把它做成产品级别的开源工具。Shannon 抓住了这个空白。

### 03 ｜典型场景和避坑

**适合谁？**

- **用 AI 编程工具（Claude Code、Cursor、Copilot）快速出活的团队** —— 代码写得快，漏洞漏的也快。Shannon 就是你的"对冲工具"。
- **小团队/创业公司** —— 没有专职安全人员，但又不想裸奔上线。花 10 分钟跑一次 Shannon，比什么都不做强太多。
- **做 CI/CD 集成的 DevOps 团队** —— 把 Shannon 塞进 pipeline，每次提交代码自动跑一遍渗透。

**不推荐给谁？**

- **大厂安全团队** —— 你们早就有成熟的商业方案了，Shannon 的定位是"人人可用的轻量级渗透"，对大厂来说太浅。
- **追求零误报的生产环境** —— 它是 AI 驱动的，不是 100% 精确。误报还会有，但比传统扫描器少得多。
- **不想面对真相的人** —— 说真的，跑完 Shannon 你可能会发现你的代码比想象中脆弱。做好准备再动手。😅

**真实场景案例：**
假设你用 Claude Code 写了一个 SaaS 应用的用户系统。Claude 帮你搞定了很多代码，但你总感觉哪里不对劲。跑一次 Shannon，它可能直接找到你的 JWT 签发逻辑里漏了签名验证——然后当场帮你生成一个伪造的管理员 token。

这种"我靠，真的能进去？"的体验，比任何安全培训都管用。

### 04 ｜上手教程

Shannon 的 README 写得很清楚，这里我摘要出最快上手的三个方式：

**方式一：直接从 PyPI 安装**

```bash
pip install shannon-lite
shannon-lite --code-dir ./my-app --target-url http://localhost:3000
```

**方式二：使用 Docker 一键运行**

```bash
docker run -v $(pwd):/app keygraph/shannon-lite \
  --code-dir /app/src \
  --target-url http://host.docker.internal:3000
```

**方式三：从源码运行**

```bash
git clone https://github.com/KeygraphHQ/shannon
cd shannon
pip install -r requirements.txt
export OPENAI_API_KEY="sk-xxx"  # 或者用 Claude API key
python main.py --code-dir ./my-app --target-url http://localhost:8080
```

几点提醒：
- **需要 API key** —— Shannon 底层用 LLM（支持 OpenAI/Claude）做代码分析和攻击决策
- **目标必须可访问** —— 它需要能物理访问到你正在跑的 Web 应用
- **建议在测试/预发布环境跑** —— 不要在线上直接用，因为它是真实 exploit
- **第一次跑别期待太多** —— 项目还在快速迭代，如果你遇到 bug，提个 issue 开发者大概率会回
