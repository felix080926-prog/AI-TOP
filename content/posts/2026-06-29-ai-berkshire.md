**大家好，我是雷达君。**

你身边有没有这样的朋友：看了巴菲特股东大会直播，买了《聪明的投资者》，甚至把芒格的"人类误判心理学"打印出来贴在桌上——但一到实际操作，还是被市场反复教育。不是理念不行，是执行太难。研报翻不完，财报看不透，DCF 模型搭到一半就想摔键盘。

更残酷的现实是，即使你用上了大模型助手，大多数 AI 投资工具依然停留在"帮你搜财报、帮你总结新闻"的层面。它们能告诉你宁德时代去年营收多少，但回答不了"如果段永平看到这个估值，他会买吗？"——因为 AI 不懂投资哲学，更不懂不同大师之间的思维冲突。

**xbtlin/ai-berkshire 正是冲着这个痛点来的。** 它不是又一个量化回测工具箱，而是一个基于 Claude Code 和 Codex 的价值投资研究框架：把巴菲特、芒格、段永平、李录四位投资大师的方法论固化为多个 AI Agent，让它们在研究同一家公司时各自独立分析、互相质疑、最终综合打分。换句话说，它让 AI 学会了"用巴菲特的逻辑去思考"，而不是简单地替你搜索。

## 01 ｜核心逻辑

ai-berkshire 的设计哲学非常清晰：**价值投资的精髓不是公式，是思考方式。** 因此它没有走传统量化路线（堆因子、跑回测），而是把每位大师的投资框架拆解成可执行的 Agent 提示链。

每个 Agent 背后是一套完整的推理路径——巴菲特的 Agent 侧重护城河、管理层质量和自由现金流折现；芒格的 Agent 专攻"误判心理学"和多学科思维模型，专门挑逻辑漏洞；段永平的 Agent 聚焦"敢为天下后"的商业模式判断和"本分"文化评估；李录的 Agent 则从文明演进和长期复利的角度审视公司。

最妙的是 **Multi-Agent 对抗机制**。四个 Agent 不是各自交完报告就完事，而是会互相"拍砖"：芒格的 Agent 会追问巴菲特的 DCF 假设是否过于乐观，段永平的 Agent 会质疑芒格认定的护城河在互联网时代是否还成立。这种"内卷式"辩论模拟了真实投资委员会的决策过程，输出的是一个经得起多方挑刺的综合判断。

技术上，整个框架跑在 Claude Code 和 Codex 上，支持命令行交互。你不需要部署复杂的后端服务，装上依赖、配好 API key 就能在终端里开始做研究了。

## 02 ｜硬核功能盘点

> GitHub 地址：https://github.com/xbtlin/ai-berkshire

- **四大师 Agent 并行研究**：同时启动巴菲特、芒格、段永平、李录四个独立 Agent，每个 Agent 按各自投资哲学进行全链路分析，从行业判断到估值模型一步到位。
- **Multi-Agent 交叉质询**：Agent 之间不是各写各的，而是主动阅读其他 Agent 的报告并提出质疑。芒格会指出巴菲特模型中的心理偏差，段永平会挑商业模式假定的漏洞，最终产出一份经过"辩论"的综合报告。
- **命令行原生体验**：完全在终端中运行，支持交互式提问。你可以随时追问"巴菲特和芒格对这个公司的分歧点在哪？"，框架会即时调度 Agent 给出回应。
- **可插拔的分析模块**：每个大师的分析框架是模块化的，你可以调整巴菲特的 DCF 参数、修改芒格的 check list，甚至添加自己定义的第五个 Agent。
- **结构化输出**：最终报告不是一段长篇随笔，而是包含财务分析、护城河评估、管理层打分、风险矩阵和综合建议的结构化文档，可以直接作为投资备忘录存档。

## 03 ｜典型场景和避坑

**典型场景一：个人投资者做深度公司研究**

以前你研究一家公司可能要花两周——翻 10 年财报、读行业研报、自己搭 DCF、翻遍雪球和公众号找不同观点。现在把股票代码丢给 ai-berkshire，四个 Agent 并行工作 15-20 分钟，给你一份包含多视角碰撞的研究报告。你可以重点看"争议点"部分——那里通常是市场定价和内在价值差距最大的地方。

**典型场景二：投研团队做路演前准备**

买方或卖方分析师在路演前需要预判所有可能被问到的刁钻问题。用 ai-berkshire 的对抗模式跑一遍目标公司，芒格 Agent 和段永平 Agent 产出的质疑清单基本覆盖了 80% 的真实路演问题。这不是替代你的专业判断，而是帮你补盲区。

**避坑指南**

- 第一，不要把它当"荐股神器"。四个 Agent 的输出质量严重依赖底层 LLM 的推理能力，目前 Claude Code 的表现最好，用其他模型可能得到泛泛而谈的结果。它帮你梳理逻辑框架，最终的投资决策必须由你自己做出。
- 第二，中文公司覆盖有待加强。框架目前对美股和港股的支持更成熟，A 股的财报格式和市场结构差异可能导致段永平和李录的 Agent 输出不够精准。如果你主要研究 A 股，建议先用茅台、招行这类信息透明度高的大票测试。
- 第三，API 成本需要提前规划。一次完整的四 Agent 研究可能消耗大量 token（尤其当你用 Claude Opus 级别模型时），建议先在小规模问题上跑通流程，确认输出质量后再放大使用。

## 04 ｜上手教程

```bash
# 1. 克隆项目
git clone https://github.com/xbtlin/ai-berkshire.git
cd ai-berkshire

# 2. 安装依赖（建议使用虚拟环境）
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# 3. 配置 API Key
# 在 .env 文件中填入你的 Anthropic API Key
echo "ANTHROPIC_API_KEY=sk-ant-xxx" > .env

# 4. 运行四大师分析（以 Apple 为例）
python run.py --ticker AAPL --agents buffett,munger,duan,li

# 5. 查看综合报告
# 报告会自动生成在 output/ 目录下
cat output/AAPL_comprehensive_report.md
```

如果你只想看某个特定视角的分析，可以指定单个 Agent：

```bash
# 只用段永平的视角看茅台
python run.py --ticker 600519.SH --agents duan
```

## 05 ｜总结

ai-berkshire 最打动人的不是它的技术有多炫，而是它抓住了价值投资最本质的东西——**独立思考 + 交叉验证**。巴菲特的"在别人恐惧时贪婪"不是一句口号，背后是需要大量扎实研究支撑的判断力。这个项目把研究流程中可标准化的部分用 AI 自动化了，让你把精力留给真正需要人的判断力的环节。

当然，它也有明显的局限。目前的版本更像一个"投研加速器"而非"投资决策引擎"——它帮你更快地看清一家公司，但不能替你决定该不该买。而且底层对 LLM 的强依赖意味着推理质量存在天花板，遇到极度复杂的商业判断时，Agent 的输出可能流于表面。

但方向是对的。当越来越多的投资决策辅助工具出现时，真正的差异点不是你用了多少数据，而是你能不能把"思考"这件事本身进行工程化。ai-berkshire 在这条路上迈出了值得关注的一步。
