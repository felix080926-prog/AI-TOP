它就是 —— **CC Switch**。

## 01 ｜核心逻辑

简单说，CC Switch 解决的问题刻在它的名字里：**Switch**（切换）。

但它的野心远不止“切”。它是一个**全家桶式的 AI CLI 配置管理中心**，把分散在你电脑各个角落的配置文件全部收拢到一个统一的桌面应用里。你只需要在 CC Switch 中完成配置，它会自动写入各个 CLI 工具对应的配置文件，实现“**单一事实源**”（SSOT）。

**中央调料台，伸手就拿。**

它基于 **Rust + Tauri 2 + React/TypeScript** 构建，所有配置本地存储在 SQLite 数据库中，采用原子写入机制。后端由 Rust 处理，利用操作系统原生 WebView 渲染，内存占用极小，可常驻系统托盘。

一句话：**不是“能切换”，而是“管理所有切换”。**

## 02 ｜硬核功能盘点

CC Switch 能让人离不开，靠的不是某一个功能，而是一整套生态级的整合力。

🔗 **GitHub 地址**：https://github.com/farion1231/cc-switch

**1. 50+ 供应商一键切换**

内置 **50 多家供应商预设**（SiliconFlow、DeepSeek、阿里百炼等），选预设、填 API Key，一键切换生效。

主界面顶部有 Claude、Codex 等选项卡，点击即可切换模型。支持热切换和冷切换。

**2. MCP 服务器统一管理**

支持 stdio/http/sse 三种 MCP 传输方式，一键同步到所有客户端。**一处编辑，多处同步。**

**3. Skills 一键安装/卸载**

自动扫描 GitHub 热门仓库，像 npm 一样批量安装、管理版本，支持 symlink 和文件复制。

**4. 用量追踪与成本管理**

内置 Token 消耗追踪、成本监控、趋势图表。系统根据配置的 Token 成本自动计算费用，实时显示。可为常用模型设置预算提醒。

**5. 本地路由 + 故障转移**

自动故障切换、熔断器、请求整流 —— API 节点挂了也能自动切换，不打断工作流。

**6. 其他亮点**

系统 Prompt 多模板管理（Markdown 实时编辑+预览）、API 速度测试与可视化、跨设备 WebDAV 云同步、配置安全备份（保留最近 10 个版本，一键回滚）、全局代理开关。

## 03 ｜典型场景和避坑

**适合谁用？**

- **日常在多个 AI 编码工具之间切换的开发者**：Codex 写代码快，Claude Code 推理强，Gemini CLI 搞多模态 —— CC Switch 让你不用关机重启就能随便切。
- **维护多套 API 配置的中转服务用户**：50+ 内置预设，支持完全自定义（Kimi、Moonshot、GLM 等）。
- **想要追踪 AI 使用成本、避免预算超支的团队或个人**：用量追踪 + 成本监控 + 报价提醒，三件套拉满。

**不推荐给谁？**

- **只用单一 AI CLI 工具、从不切换的用户**：装了意义不大。
- **完全不熟悉 CLI 环境、命令行基本不太碰的纯业务用户**：它毕竟是为 AI CLI 工具量身打造的桌面应用。
- **追求绝对零配置、不想装任何额外软件的用户**：多一个软件就是多一个。

## 04 ｜ 5 分钟快速上手

**安装**（根据你的系统选择）：

- **macOS（最推荐）**：`brew tap farion1231/ccswitch && brew install --cask cc-switch`
- **Windows**：下载 `.msi` 安装包或 Portable ZIP 解压即用
- **Linux**：支持 `.deb`、`.rpm`、`.AppImage`，Arch 用户可 `paru -S cc-switch-bin`

**首次使用**：

1. 启动后，CC Switch 会自动检测已安装的 CLI 工具并尝试导入现有配置
2. 点击右上角“Add Provider”，从 50+ 内置预设中选择，或手动添加自定义供应商
3. 填写 API Key 和 Base URL，点击 Enable 即可生效

**热切换**：主界面顶部三大选项卡，点击即切换，无需重启当前工作流。
