**大家好，我是雷达君。**

你有没有经历过这样的场景：想搭一个 3D 游戏原型，光是搭场景、写脚本、调材质、跑测试——一套流程走下来，半天就没了。

更要命的是，你在 Unity 编辑器里来回点菜单、拖资源、写 C#代码，每一步都是重复劳动。一个简单的“给这个 Cube 加个刚体”，你得先选中物体，点 Add Component，搜 Rigidbody，再点确认——四步操作，一分钟。

如果有一个 AI 助手，你只需要说一句“**在原点创建一个 Cube，加上刚体组件**”，它就在编辑器里帮你把活干完了呢？

今天雷达君要安利的这个开源项目，干的就是这件事。它是一个基于**MCP（模型上下文协议）** 的桥梁，把 Unity 编辑器和 AI 助手彻底打通——Claude、Cursor、VS Code、Codex、Gemini CLI，任何支持 MCP 的客户端，都能直接“指挥”你的 Unity 编辑器干活。

它就是——**Unity MCP**。

## 01 ｜核心逻辑：把 Unity 编辑器变成 AI 的“提线木偶”

Unity MCP 的核心逻辑不复杂，但它做的事情很有想象力：**让 AI 像操作一个遥控玩具一样操作 Unity 编辑器。**

传统的 AI 辅助开发是什么样？AI 给你生成一段 C#代码，你自己复制粘贴到 VS Code 里，再切回 Unity 编辑器编译、运行、调试。来回切换窗口，打断思路，效率大打折扣。

Unity MCP 换了一套打法。它通过**MCP 协议**在 AI 客户端和 Unity 编辑器之间建立了一条双向通道。AI 不再是“给你写代码让你自己贴”，而是直接**在编辑器里执行操作**——

- AI 说“创建一个 Cube”，编辑器里真的出现一个 Cube
- AI 说“给这个物体加上 Rigidbody”，组件真的被挂上了
- AI 说“运行测试”，测试真的开始跑

整个过程你不需要离开 AI 聊天界面。Unity 编辑器就像被 AI 远程操控了一样，你在对话里发号施令，编辑器那边自动执行。

**一句话：你不是在用 AI“辅助”开发，你是在用 AI“指挥”开发。**

## 02 ｜硬核功能盘点

> 🔗 **GitHub 地址**：https://github.com/CoplayDev/unity-mcp

Unity MCP 目前提供了**47 个聚焦的 MCP 工具入口点**，覆盖了 Unity 开发的方方面面：

### 1. 场景与 GameObject 管理：说一句话，搭一个场景

这是 Unity MCP 最直观的能力。你可以用自然语言直接操控场景：

- “在(0,0,0)位置创建一个 Cube”
- “把主摄像机移到(5,3,-10)”
- “创建一个球体，半径为 2，红色材质”
- “选中场景里所有带有 Rigidbody 的物体”

不需要点菜单、不需要拖拽、不需要写编辑器脚本。一句话，AI 帮你搞定。

### 2. C#脚本编辑与编译：让 AI 当你的“编程副手”

Unity MCP 不仅能操作场景，还能直接读写和修改 C#脚本：

- “创建一个 PlayerController 脚本，包含 WASD 移动逻辑”
- “给这个脚本添加一个 Jump 方法”
- “修复这个脚本里的编译错误”

AI 生成的代码直接写入项目，Unity 自动编译，你在编辑器里就能立刻看到效果。

### 3. 资源管理与导入：资产操作自动化

管理游戏资源是 Unity 开发里最繁琐的工作之一。Unity MCP 提供了完整的资源管理工具：

- “把这张图片导入为 Sprite”
- “创建一个材质，颜色为蓝色，金属质感”
- “删除项目中所有未使用的资源”

### 4. 测试与性能分析：让 AI 帮你跑测试、查性能

Unity MCP 还集成了测试和性能分析工具：

- “运行所有单元测试”
- “对当前场景做性能分析”
- “检查内存泄漏”

你在对话里下指令，AI 在编辑器里执行，结果直接返回给你。

### 5. 工具分组（v10.0.0 新增）：按需启用，不臃肿

最新发布的**v10.0.0**版本引入了**工具分组（Tool Groups）** 功能。你可以按需启用特定领域的工具集——VFX、动画、UI、测试等——避免一次性加载全部工具导致编辑器卡顿。这个设计让 Unity MCP 从“功能多”升级到了“功能多且精”。

### 6. 多 Unity 实例支持：同时操作多个项目

如果你同时打开多个 Unity 项目，Unity MCP 支持**多实例路由**。你可以指定 AI 操作哪一个编辑器实例，不会混淆。

### 7. 全客户端兼容：Claude、Cursor、VS Code、Gemini，全部打通

Unity MCP 基于 MCP 标准协议构建，兼容任何支持 MCP 的客户端：

- **Claude Desktop & Code**
- **Cursor**
- **VS Code**
- **Windsurf**
- **Cline**
- **Gemini CLI**

你习惯用哪个 AI 工具，就用哪个“指挥”Unity。

## 03 ｜快速上手指南（5 分钟）

Unity MCP 的安装流程非常清晰，官方提供了两种主流方式。

### 方式一：Unity Package Manager 安装（推荐）

**第 1 步：在 Unity 中安装 MCP 包**
打开 Unity，进入`Window → Package Manager`，点击左上角的“+”号，选择**Add package from git URL**，输入：

```
https://github.com/CoplayDev/unity-mcp.git?path=/MCPForUnity#main
```

想锁定特定版本，把`#main`换成`#v10.0.0`即可。

**第 2 步：配置 MCP 客户端**
安装完成后，进入`Window → MCP for Unity → Configure All Detected Clients`。Unity MCP 会自动检测你电脑上已安装的 MCP 客户端（Claude Desktop、Cursor 等），一键完成配置。

**第 3 步：开始指挥**
在 AI 客户端里输入：

```
“在原点创建一个Cube，并添加Rigidbody组件。”
```

Unity 编辑器里会立刻出现一个带刚体的 Cube。

### 方式二：OpenUPM 安装（适合包管理爱好者）

如果你习惯用 OpenUPM 管理 Unity 包，也可以执行：

```
openupm add com.coplaydev.unity-mcp
```

## 04 ｜典型场景和避坑

**适合谁用？**

- **Unity 游戏开发者**：用自然语言加速场景搭建、脚本编写和测试流程，把重复劳动交给 AI
- **独立游戏开发者**：一个人就是一个团队，AI 帮你分担场景、脚本、资源管理的体力活
- **想用 AI 辅助 Unity 开发但不想频繁切换窗口的开发者**：在 AI 对话里完成所有操作，不用来回切编辑器
- **游戏开发教学/演示场景**：用自然语言实时演示 Unity 操作，学生看得更直观

**不推荐给谁？**

- **不用 Unity、不做游戏开发的人**：这个项目是 Unity 专属，其他场景用不上
- **完全不用 MCP 兼容客户端（Claude/Cursor 等）的用户**：Unity MCP 需要 MCP 客户端作为“指挥端”

## 05 ｜总结

Unity MCP 解决了一个很朴素但很真实的问题：**Unity 编辑器操作太繁琐了。**

创建物体要四步，加组件要三步，写脚本要切窗口，跑测试要点菜单——这些重复劳动占据了开发者大量的时间。Unity MCP 用 MCP 协议把 Unity 编辑器和 AI 助手彻底打通，让 AI 成为你的“Unity 操作员”。

47 个工具入口点、场景管理、脚本编辑、资源操作、测试运行、性能分析、工具分组、多实例支持——这些能力让 Unity MCP 从一个“玩具”变成了一个真正能提升开发效率的生产力工具。
