## 01 ｜核心逻辑：从“模型能力”到“工具主权”

现在的 AI 已经足够聪明，但它缺乏**行动的触手**。

**Awesome Codex Skills 的逻辑非常硬核：** 它把上百个第三方服务和本地工具的操作流程，全部封装成了 AI 可以直接识别并调用的 **YAML/JSON 技能描述文件**。
它不是简单的 API 调用，而是带逻辑、带权限、带反馈的**原子化技能**。

- **即插即用：** 你不需要写复杂的代码去教 AI 怎么用 Slack，直接加载对应的 Skill 包即可。
- **生态互通：** 它能让不同的 Agent 框架（如 Multica、Claude Code 等）共享同一套技能库。

## 03 ｜场景演示：它是如何实现“自动化闭环”的？

想象一下，你正在处理一个紧急的 Bug 修复。

1. **加载技能：** 你给 AI 挂载了 `GitHub`、`Linear` 和 `Slack` 三个技能包。
2. **派发任务：** 你说：“把 Linear 里编号 102 的 Bug 修复了，提个 PR，然后在 Slack 里同步给测试组。”
3. **自主执行：**

- AI 调用 `Linear` 技能获取 Bug 详情。
- 在本地修改代码并测试。
- 调用 `GitHub` 技能提交 Commit 并创建 PR。
- 调用 `Slack` 技能向频道发送通知。

4. **结果：** 你全程在喝咖啡，AI 已经完成了跨平台的**整套链路操作**。
