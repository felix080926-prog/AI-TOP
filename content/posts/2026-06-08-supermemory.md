**大家好，我是雷达君。**

**深夜加班，你问 AI“还记得上周那个项目的核心需求吗？”**
它沉默三秒，然后像第一天上班一样，礼貌地回复：“好的，请告诉我项目背景。”

你的 AI 能写代码、能画图、能打败 90%的人类做题家，但它是世界上最健忘的物种。因为大模型天然就没有长期记忆。

这就是今天雷达君要深度安利的项目——**Supermemory**。
它不是你的又一个笔记软件，它是你 AI 的**海马体**。给所有 AI 工具装上同一个大脑，让它们跨越会话、跨越工具，记住关于你的一切。

它就是为 AI“消除失忆”而生——**Supermemory**。

## 01 ｜核心逻辑：AI 的“海马体”

当你在 Claude Chat 聊代码，在 Cursor 写代码，再跑去 Notion 记笔记，这三者之间有一个巨大的逻辑断层：**信息是割裂的**。

Supermemory 把这三者统一起来，构建一个**持久化、跨应用、自学习的统一记忆层**。它像一个外挂硬盘，所有 AI 都只读这一个盘子。

你不再需要反复提醒 AI“我是谁”；你收藏的推特、PDF、甚至和朋友的聊天，都能成为 AI 的上下文。它采用了“智能记忆管理”机制，自动索引、提取事实、处理矛盾信息、甚至具备遗忘功能，就像一个真实的大脑突触连接。

**一句话总结：别人的 AI 聊完就忘，你的 AI 过目不忘。**

## 02 ｜硬核功能盘点

> GitHub 地址：https://github.com/supermemoryai/supermemory

Supermemory 不仅是一个 App，更是一套**统一的记忆基础设施（Memory API）**。它的生态建设完整度令我惊讶，核心功能远超普通笔记软件。

**1. 基准测试屠榜：AI 记忆赛道的性能怪兽**

作为记忆引擎，Supermemory 拿到了硬核的成绩单。它在 LongMemEval、LoCoMo、ConvoMem 这三大权威 AI 记忆基准测试中均排名第一。这意味着无论是长对话中的事实提取精度，还是长时间跨度的记忆检索能力，它都经过了最严格的学术验证。加上端到端加密与自托管选项，数据权完全掌握在你手中。

**2. 百川归海：数据摄取“一体机”**

构建第二大脑最痛苦的是“录入”。Supermemory 不仅是笔记软件，更是一个强大的信息摄取中心。通过**Chrome 浏览器插件**，你可以一键把 Twitter、网页、PDF 甚至视频扔进去。通过**Raycast 扩展**，Mac 用户键盘快速记录灵感。它支持连接 Google Drive、Notion 等第三方服务，自动同步云文件。你还可以上传代码、图片（OCR）、视频（转录），它全部通吃。

**3. 万物互联：让每个 AI 都拥有“全局记忆”**

这是 Supermemory 最有想象力的能力——**跨平台记忆共享**。它提供了一个插件市场，支持 Claude Code、OpenClaw、OpenCode、Hermes 等主流 AI 编程工具。它还提供**MCP 服务器**，接入 Claude Desktop 或 Cursor 后，这些工具将共享同一个记忆库。

你甚至可以把它当作文本知识库，直接作为 AI Agent 的 RAG 后端。作为一个提供**RESTful API**的记忆引擎，你不再需要自己配置繁琐的向量数据库、嵌入管道和分块策略。

**4. 自然语言交互：像人一样“想”和“问”**

信息检索不是靠关键词，而是靠**自然语言提问**。你可以问：“帮我总结上周保存的 AI 论文核心观点”，或者“找一下上次和客户讨论的关于预算的那份 PDF”。它的“Canvas”功能支持在 Markdown 编辑器里通过 AI 辅助整理知识，告别传统笔记的低效标签管理。

## 03 ｜横向对比：为什么是 Supermemory？

AI 记忆赛道并非 Supermemory 一枝独秀，但它的差异化定位非常清晰：

| 维度             | **Supermemory**                 | **Mem0**         | **Notion AI**            |
| :--------------- | :------------------------------ | :--------------- | :----------------------- |
| **核心定位**     | 统一记忆层 + API                | 轻量级记忆 API   | 知识库/文档管理          |
| **适用对象**     | AI 应用开发者 + 个人            | AI 应用开发者    | 个人/团队（非开发者）    |
| **记忆持久性**   | **跨会话、跨应用**              | 跨会话           | 受限于工作区，需手动整理 |
| **多模态支持**   | PDF、图片、视频、代码           | 文本为主         | 文件附件，非深度语义检索 |
| **接入 AI 工具** | 原生插件（Claude Code 等）+ MCP | 需自行开发集成   | 仅限 Notion 内部 AI      |
| **基准测试**     | **三项榜单第一**                | 尚无公开领先数据 | 不适用                   |

Supermemory 在**持久化、多模态摄取**和**生态集成（MCP/API）** 三个维度上同时发力，无论是个人快速上手还是企业级 API 调用，都提供了标准化的方案。

## 04 ｜典型场景和避坑

**适合谁用？**

- **深度依赖 AI 编程（Claude Code/Cursor），且项目逻辑复杂的技术人**：让 AI 记住项目全貌和你的编码偏好，上下文窗口不够用？Supermemory 帮你“外挂”记忆。
- **跨平台信息量爆炸的创作者/研究员**：浏览器、Notion、PDF、推文全打通，不再担心知识碎片化。
- **想构建个性化 AI 应用，但不想碰底层复杂记忆栈的开发者**：一行代码集成 Memory API，你只负责业务逻辑，它解决“记忆力”问题。

**不推荐给谁？**

- **只用纯对话服务（如 ChatGPT 网页版），且不在意记忆短暂的用户**：加装 Supermemory 属于杀鸡用牛刀。
- **极度追求离线、完全本地化存储且不接受云端同步的“隐私洁癖”**：虽然 Supermemory 支持自托管，但多数便捷功能（浏览器插件同步等）依赖云端服务，完全离线使用会大幅降低体验。
- **完全无代码基础，只想要一个“完美离线笔记软件”的用户**：可以仅用它的 Web App，但如果在自托管或 API 集成时缺乏技术支撑，会遇到安装门槛。

## 05 ｜快速上手指南：构建你的第二大脑

上手 Supermemory 有多种途径，我已经梳理出从“零门槛使用”到“企业级集成”的全路径：

**方式一：开箱即用（最推荐新手）**

直接访问`https://app.supermemory.ai`注册免费账号。登录后，立即在 Web 界面中保存内容。下载 Chrome 插件，在 Twitter 阅读文章时点击保存；访问 Notion 或 Google Drive，通过连接器一键导入。右上角 Chat 界面提问，体验 RAG 检索能力。

**方式二：为 Claude Code 装上大脑**

克隆仓库：`git clone https://github.com/supermemoryai/claude-supermemory`。按照 README 配置环境变量（主要是 API Key 和数据库）。在 Claude Code 中启用插件。以后每次对话，AI 都能自动访问你在 Supermemory 中存储的所有知识。

**方式三：开发者自托管**

对希望完全掌控数据的用户，Supermemory 提供完善的自托管指南：

```bash
git clone https://github.com/supermemoryai/supermemory.git
pnpm install
cp .env.example .env
pnpm dev
```

浏览器访问`http://localhost:3000`即可进入本地实例。

**方式四：使用 Memory API（适合企业级 AI 应用）**

对于希望在 AI 应用中集成记忆能力的企业用户，可通过 API 快速接入：

```bash
curl -X POST https://api.supermemory.ai/v1/memory \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"content": "用户偏好使用TypeScript和React Query", "user_id": "user_123"}'
```

一次调用，永久记忆。不再需要单独维护向量数据库、嵌入管道和分块策略。
