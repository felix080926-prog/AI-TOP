**大家好，我是雷达君。**

这两年用 AI 写前端的体验，说好听点叫"高效"，说难听点就是"开盲盒"。你让它生成一个 SaaS 落地页，它给你一个渐变紫背景 + 毛玻璃卡片 + 3D 漂浮按钮的缝合怪；你让它做一个极简博客，它依然给你渐变紫背景。不同需求出来的页面长成一个妈生的，区别只是标题换了个字——这就是 AI 生成代码的"模板同质化灾难"。

更可怕的是，你反复调 prompt、加约束、甚至写了几百字的角色设定，AI 还是会在你不注意的角落偷偷塞进一个发光边框或者无意义的渐入动画。这些被社区戏称为"AI slop"的视觉垃圾，本质上是因为大模型在训练时被喂了太多模板网站的代码，导致它们把"花哨"误认成了"好看"。

直到今天，一个叫 Hallmark 的开源项目站了出来——不是又教 AI 怎么写代码，而是给 AI 上了一堂设计课。它像一个坐在 AI 旁边的艺术总监，在代码输出之前先过一道审美审查：这个字体斜了吗？这个渐变动画删掉。那个"10 倍提效"的数据是你编的吧？打回去重写。

Hallmark 由知名开源作者 Nutlope 开发，发布不到三个月就斩获 11K Star，今天单日增长 3,372 Star。值得一提的是，它来自 Together AI 团队，核心思路是把人类设计师对"AI 味道太重"的共识提炼成一套可执行的规则，然后硬编码进 AI 编程工具的工作流里——不是建议，是规则。

## 01 ｜核心逻辑

Hallmark 的底层逻辑并不复杂，但非常巧妙。它本质上是一套"反 AI 套路"的 checking 机制，由三个关键环节组成：结构多样性引擎、57 道品味关卡，以及输出前的自我评分机制。

先说结构多样性。大多数 AI 生成的页面都是同一个骨架——Hero 大图 → 三个 feature 卡片 → CTA 按钮 → Footer。Hallmark 强制要求每次生成都必须选择不同的"宏观结构"，同样两个项目需求，一个可能走杂志式排版（大标题 + 侧栏导航），另一个可能走产品叙事流（单屏滚动 + 逐段展开）。它维护了一个结构轮换记录，确保同一项目里不会连续两次产出雷同的页面骨架。

再说品味关卡。Hallmark 内置了 57 条"slop-test gate"——每一条都是社区公认的 AI 代码坏味道。比如 Gate 46 禁止编造数据（"10 万+用户信任我们"这种万能废话），Gate 38a 禁止斜体标题（AI 最爱把标题设成 italic，但专业设计师几乎从不用斜体做 heading），Gate 47 禁止画假浏览器框（AI 特别喜欢给截图套个假的 Chrome 地址栏）。任何一条没通过，代码就被打回去重写。

最后是输出前的自我评分。在代码最终交给用户之前，Hallmark 会给自己打分——哲学性、层级感、执行精度、特异性、克制度、多样性，六个维度各 1-5 分。任何一个低于 3 分就自动触发重写。这个机制有效防止了 AI 在生成长代码时"后半段随便凑合"的毛病。

## 02 ｜硬核功能盘点

> GitHub 地址：https://github.com/Nutlope/hallmark

- **20 套预设主题 + 自定义路线**：Hallmark 内置了 Bubble、Cobalt、Lumen、Carnival、Garden 等 20 个具名主题，覆盖从极简到编辑风格的完整光谱。如果你的需求比较独特，它还会自动切换到 Custom 模式，从 OKLCH 色板和自由字体配对阵中生成一套独一无二的视觉系统。两套模式共享同一套 57 道品味关卡，Custom 模式下不会因为"自由发挥"而放松标准。

- **四大动词：不止生成，还能审计、重构、提取**：这是 Hallmark 真正超出普通代码模板的地方。默认模式是构建新 UI；`audit` 能扫描已有代码并给出反模式评分清单；`redesign` 保留内容和信息架构，只翻新视觉层；`study` 更惊艳——给它一个截图或 URL，它能提取出该设计的 DNA（宏观结构、字体搭配、色彩锚点），生成一份可移植的 design.md，然后你用这份 DNA 去生成全新页面。这意味着你可以让 AI 学习 Notion 的排版，用来生成你自己的产品主页。

- **58 条反 AI 套路检测**：上面提到的 57 道品味关卡加上一条组件级别的状态完整性检查。组件必须覆盖全部 8 种状态（默认、悬浮、聚焦、按下、禁用、加载中、错误、成功），每个状态都有可视化差异，不能只靠改一个 opacity 敷衍。

- **安装极度简单**：一条命令 `npx skills add nutlope/hallmark` 就能在 Claude Code、Cursor、Codex 中启用。也支持手动复制 SKILL.md 到对应目录。MIT 开源，零成本接入。

## 03 ｜典型场景和避坑

**典型场景一：产品 Landing Page 从 0 到 1**

你有一个 SaaS 产品，需要快速出一个专业级的落地页。以前的做法是——打开 Cursor，描述需求，AI 吐出一个中规中矩但"AI 味"很重的页面，然后你花两小时手动删渐变、调间距、换字体。用了 Hallmark 后，同样的 prompt 下，AI 会自动选择对应主题（比如 SaaS 类自动匹配 modern-minimal 路线），输出前跑完 57 道关卡，最终给你的页面结构清晰、排版专业，而且不会在角落偷偷放一些"自信到可疑"的数据。

**典型场景二：接手旧项目，视觉重构**

你接手了一个两年前的老项目，代码能跑但 UI 惨不忍睹。用 `hallmark redesign`，它不会动你的路由、组件逻辑和业务代码，只重新设计视觉层。新的设计还会写入一个 `design.md` 文件，后续整个团队的 AI 工具都按统一标准出活，避免一个人一个风格。

**避坑指南**

- 第一，Hallmark 是设计技能，不是万能代码生成器。它管的是"代码长什么样"，不管"代码能不能跑"。如果你的需求是复杂的交互逻辑（比如一个拖拽排序表格），Hallmark 不会插手——它的范围严格限定在视觉和排版层面，别指望它帮你写业务逻辑。

- 第二，注意动词选择。很多人装了就无脑用默认模式，其实 `audit` 和 `study` 才是真正提效的地方。在做新页面之前先 `study` 一个你喜欢的设计参考，提取 DNA 后再生成，效果比直接生成好一个数量级。

- 第三，自定义主题别滥用。Custom 模式非常强大但消耗更多 token。普通需求交给默认的 20 套主题就行，它们已经覆盖了绝大多数场景。只有在你明确说了"品牌色是 #某颜色"或者"要某种特定氛围"时，再让 Custom 介入。

## 04 ｜上手教程

**安装**

```bash
npx skills add nutlope/hallmark
```

执行后，Hallmark 会自动注册到你的 AI 编程工具中。以 Claude Code 为例，安装完成后直接在同个终端启动 Claude Code，Hallmark 的规则就已经生效了。

**基础用法**

启动你的 AI 编程助手，直接像往常一样描述需求：

> 帮我做一个个人博客的首页，极简风格，白色背景，黑色文字，要有文章列表和关于我区块。

Hallmark 会自动介入：检测信号 → 匹配主题（这里会走 editorial 路线）→ 选择宏观结构 → 应用反套路关卡 → 自我评分 → 输出。

**审计已有代码**

```bash
# 在 AI 编程助手中输入
hallmark audit src/pages/Landing.tsx
```

你会得到一份按严重程度排序的问题清单，比如："第 42 行：CTA 按钮使用了 box-shadow: 0 20px 80px rgba(...) 发光效果，建议改为 subdued 投影。"

**从参考设计提取 DNA**

```bash
hallmark study https://linear.app
```

它会分析 Linear 的排版、配色、间距、CTA 风格，提取出一份 design.md。然后你可以说 "用这份 DNA 重建我的项目首页"，一键对齐业界最顶尖的设计系统。

**注意**：目前 Hallmark 主要支持 Claude Code、Cursor 和 Codex。VS Code + Copilot 用户建议使用 Cursor 或直接复制 SKILL.md 到项目规则文件。

## 05 ｜总结

Hallmark 解决了一个被很多人忽视但影响巨大的问题：AI 生成代码的审美一致性。它不是在创造新的设计理论，而是把专业设计师已经达成共识的"什么不该做"编码成了自动化检查。57 道关卡、6 维度自我评分、20 套预设主题——这些数字背后是一个朴素的信念：代码可以自动化生成，但品味不能自动化妥协。

当然，Hallmark 不是银弹。它管的是视觉和排版，管不了复杂交互逻辑；它的 Custom 模式虽然强大但消耗 token 较多；它对中文排版的支持还在早期阶段（斜体检测 gate 38a 在中文语境下偶有误判）。但这些局限不影响它在自己领域里的统治力——如果你受够了 AI 代码的"渐变紫恐惧症"，Hallmark 是目前最好的解药。

Nutlope 这个作者也值得关注。他之前出过 anthena、llama-tutor 等多个高质量开源项目，风格鲜明：不追求大而全，而是盯住一个具体痛点做深做透。Hallmark 延续了这个路线，并且这一次，它打的痛点是每个用 AI 写代码的人都绕不过去的坎。
