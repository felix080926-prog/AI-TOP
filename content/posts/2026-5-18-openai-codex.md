2018 年，我花了一整天折腾 Vim 配置，只为让编辑器能自动补全括号。

2022 年，GitHub Copilot 出来了，我对着它能自动生成整个函数这件事傻笑了半小时。

**2026 年，OpenAI 直接把编程助手扔进了终端——不是插件，不是网页，是 `npm install -g @openai/codex` 就能用的东西。**

这家把 ChatGPT 卖给全世界、把 GPT API 做成基础设施的公司，现在告诉你：我们做了个开源的工具，装好之后你在终端里输几个字，它就能帮你写代码、改代码、修 bug、甚至跑测试。

而且，它不叫 ChatGPT，不叫 Copilot。

它叫 Codex。

等等——Codex 不是那个模型的代号吗？

没错。但这次，它是一个开源产品的名字。

---

## 01 ｜核心逻辑

这事儿得从名字说起。

2021 年，OpenAI 发布了一个叫 Codex 的模型——它是 GPT 的变体，专门用来写代码。这个模型后来成为了 GitHub Copilot 的底层引擎，养活了无数开发者的"Tab 键"。

2025 年底，OpenAI 重新启动了"Codex"这个名字，但这次**不再是卖模型**。

这次是一整套的产品线：

- **Codex CLI**：终端里的 AI 编程助手（开源）
- **Codex IDE**：VS Code / Cursor / Windsurf 插件
- **Codex Desktop**：桌面端独立应用
- **Codex Web**：网页版（chatgpt.com/codex）

你可能会问：这不就是个 copilot 的换皮版吗？

还真不是。

本质区别在于：**Codex CLI 的核心假设是「最好的编程界面就在终端里」，而不是「你得到一个编辑器里才能用 AI」。**

说白了，OpenAI 把赌注押在了一个方向：未来真正的 AI 编程不是"在编辑器里按 Tab 补全"，而是"在终端里跟 AI 对话，让它帮你完成整个任务"。

这个理念听起来简单，但它跟现在所有主流的 AI 编程助手——Cursor、Copilot、Windsurf——走的都不是一条路。

那些产品做的是：**增强编辑器**。

Codex 做的是：**替代你敲命令**。

更骚的是，Codex CLI 跑在**你自己电脑上**，不依赖云端。你在终端里说"帮我重构这个 API 路由"，它就直接在本地读文件、改代码、跑测试，然后把结果告诉你。

## 02 ｜硬核功能盘点

> 🔗 GitHub 地址：https://github.com/openai/codex

### 功能一：纯终端交互
**一句点评：「终端就是 IDE」的最激进实践。** 不需要打开任何编辑器，`codex` 命令下去，直接在 terminal 里跟你对话。对于常年泡在终端里的开发者来说，这体验比切到浏览器或者 VSCode 快太多了。最牛的是它能看到你当前目录的文件结构，不用你手动粘贴代码。

### 功能二：本地 Agent 能力
**一句点评：它不是一个聊天框，它真的能「做事」。** 告诉它"找到所有超过 100 行且没有单元测试的函数，给它们加上测试"，它就主动去翻文件、分析代码、写测试文件、然后跑一遍看看能不能通过。每一步你都能在终端里看到实时日志，中途可以打断和修改。

### 功能三：多平台原生安装
**一句点评：一条命令搞定，对开发者友好到犯规。** `npm install -g @openai/codex` 或 `brew install --cask codex`，装完直接用。macOS（Intel 和 Apple Silicon）、Linux（x86_64 和 arm64）全覆盖。基本上你能想到的开发环境都支持了。

### 功能四：IDE 插件加持
**一句点评：给那些「离不开编辑器 UI」的人留了一扇门。** 如果你还是想要编辑器里的 AI 体验，Codex 也提供了 VS Code、Cursor、Windsurf 的 IDE 插件版本。终端派用 CLI，UI 派用插件，各取所需。

### 功能五：ChatGPT 账号打通
**一句点评：云端的算力 + 本地的操作，算是性价比最高的组合了。** 用 ChatGPT Plus/Pro 账号登录，就能直接使用。如果你有 OpenAI API 的 key，也可以通过 API key 方式认证。等于说，你把 ChatGPT 的能力直接搬到了自己的终端里，不用来回复制粘贴代码片段了。

### 功能六：Apache-2.0 开源
**一句点评：OpenAI 终于学会「用开源换生态」了。** 代码完全开源，Apache-2.0 协议。这意味着你可以 fork 它、改它、甚至基于它做自己的产品（前提是遵守协议）。这对社区生态的拉动效果，可能比功能本身还要大。

## 03 ｜典型场景和避坑

### ✅ 适合谁？

**日常开发重度终端用户**
如果你每天都在终端里敲命令、用 vim/neovim/nano、ssh 到服务器上写代码——Codex CLI 就是为你量身打造的。你甚至不需要离开终端界面。

**需要快速原型验证的开发者**
想快速试个新 API、写个脚本处理数据、搭个小服务？直接跟 Codex 说需求，它帮你把代码搭起来，你改一改就能用。比从栈溢出复制粘贴还快。

**做代码审查和重构的人**
"把这个模块的 callback 全部改成 async/await"——这个需求你说给 Codex，它真能一行一行帮你改完。对于代码迁移、重构这类重复性强的工作，效率提升非常明显。

### ❌ 不推荐给谁？

**完全依赖 GUI 的开发者**
如果你不习惯终端操作、全靠 IDE 的图形界面写代码，那 Codex CLI 的核心体验对你就没什么吸引力。建议直接上它的 IDE 插件版。

**写非常小众语言的人**
Codex 在 Python、JavaScript/TypeScript、Go、Rust、Java 等主流语言上表现很好，但如果你用的是某种冷门 DSL 或者自定义语言框架，它的表现可能会打折扣。

**还在观望 AI 编程的保守派**
如果你对 AI 生成的代码天然持怀疑态度，觉得"AI 写的代码不敢用"，那现在还不是你的入场时机。Codex 不会帮你改变这个看法——它只是让 AI 写得更快。

## 04 ｜上手教程

**前置条件：** 一个 ChatGPT 账号（Plus/Pro 实测体验最佳）或者 OpenAI API Key。

### 方式一：npm 安装（推荐）

```bash
npm install -g @openai/codex
```

装完之后，在任意项目目录下直接运行：

```bash
cd your-project
codex
```

它会让你登录 ChatGPT 账号。登录完就能用了。

### 方式二：Homebrew 安装

```bash
brew install --cask codex
codex
```

### 方式三：下载二进制文件

去 GitHub Releases 页面下载对应平台的压缩包，解压后放到 `$PATH` 里即可。

https://github.com/openai/codex/releases/latest

### 试试它能不能干活

登录之后，找个代码目录，试试下面几件事：

```bash
# 让它解释当前目录
codex "这个项目是做什么的？"

# 让它帮你写代码
codex "帮我写一个 Express 中间件，用来记录所有 API 请求的耗时"

# 让它帮你改代码
codex "找到所有使用 var 声明的地方，改成 const 或 let"

# 让它帮你找 bug
codex "这个项目里有没有明显的逻辑错误？"
```

**小技巧：** 如果你想让 Codex 帮你做一个比较复杂的任务，建议先切到一个 Git 分支上——这样试错了也不影响主分支代码。

## 05 ｜总结

OpenAI Codex 让我想起 2022 年底 ChatGPT 刚发布时的感受：**你知道它会改变点什么，但你没想到它会来得这么快。**

Codex 的核心理念很简单——编程的终极形态不是 IDE 里的几个 Tab 键，而是你告诉电脑你想要什么，它帮你搞定。终端就是最好的交互界面，因为它足够底层、足够透明。

当然，它还不是完美的。对于复杂项目的上下文理解、长时间任务的稳定性、以及一些冷门场景的适配，都还有优化空间。但这不妨碍它成为 2026 年至今我最想让开发者朋友去试试的开源项目。

**一句话总结：如果你还没有试过终端里的 AI 编程助手，codex 可能是你入坑的最好选择。**
