大家好，我是雷达君。

2019 年，柏林的一个小团队发布了一个开源项目，口号很简单：**"Zapier 的开源替代品。"**

那时候没人当回事。Zapier 如日中天，谁需要一个自己搭的轮子？

七年后的今天，这个项目有了 19 万 Star，日增近 800，名字叫 **n8n**。更关键的是，它早已不是什么"替代品"了——它变成了一个原生 AI 的工作流平台，把可视化拖拽和 AI Agent 揉在了一起，走了一条没人走过的路。

这事挺有意思的。我们聊聊。

## 01 ｜核心逻辑

n8n 最让我服气的地方，不是它功能多，而是**它在正确的时间做了正确的事**。

工作流自动化这个赛道人太多。Zapier、Make（前 Integromat）、IFTTT，全是老玩家。n8n 早期能活下来，靠的是两个杀手锏：**自部署**和**Fair-Code 许可证**。

什么意思？

你的数据不必经过第三方。你可以在自己的服务器上跑 n8n，所有工作流的数据交互都在你眼皮底下。这对金融、医疗、或者任何一个有合规要求的公司来说，几乎是刚需。

但真正让它起飞的是 **AI 的到来**。

2024-2025年，n8n 团队做了一个很大胆的决定：**不把 AI 当外挂，而是把 AI 融进内核**。他们推出了 AI Agent 节点、AI 转换节点、向量数据库支持——这些不是"顺便塞进去"的功能，而是跟普通工作流节点平起平坐的一等公民。

结果就是：你现在打开 n8n，可以拖一个 AI Agent 节点，连上你的 Slack/邮件/数据库，让它自动处理客服工单；也可以拉一个 LLM 节点，让它帮你批量分析用户反馈，然后把结果写回飞书表格。

**不是把 AI 加到工作流里，而是工作流为 AI 而生。** 这是本质区别。

## 02 ｜硬核功能盘点

> 🔗 GitHub 地址：https://github.com/n8n-io/n8n

### 功能一：AI Agent 节点（这个最骚）

n8n 的 AI Agent 节点是个集大成者。你可以给它配置：
- 使用的 LLM（OpenAI、Claude、Ollama 本地模型，随便选）
- 可以调用的工具（你自己定义，也可以用它内置的 400+ 集成）
- 记忆机制（短期/长期记忆，让 Agent 能记住上下文）

**为什么好：** 别的平台让你写代码调 Agent，n8n 让你拖。但又不是傻瓜式的拖——你仍然可以写自定义代码来精细控制。这种"给开发者留后门"的设计，让它既有速度又有深度。

### 功能二：400+ 原生集成

从 Slack、Gmail、GitHub 到飞书、钉钉、企业微信，基本上你日常用的 SaaS 全有。而且每个集成都是原生节点，不是那些通用 Webhook 糊弄你的。

**为什么好：** 你不需要学每个平台的 API。拖一个节点，填个 Token，完事。

### 功能三：自部署 + 云部署双模式

你可以用 Docker 一键部署在自己服务器上，也可以直接用 n8n Cloud 付费省事。

**为什么好：** 企业级场景里，"数据不出内网"是红线。n8n 的自部署模式是它干掉 Zapier 的最大武器——Zapier 的 Enterprise 套餐贵得离谱，而 n8n 你自己搭一分钱不花。

### 功能四：表达式引擎 v2

2026 年的新版本升级了表达式引擎，支持更复杂的变量处理和条件逻辑。写起来跟写代码差不多，但可视化。

**为什么好：** 工作流最怕的就是"做一半发现逻辑不够灵活"。表达式引擎让你可以在拖拽界面里直接写类似 `{{ $json.data.status === 'pending' ? 'wait' : 'process' }}` 的条件判断，不用额外拉一堆 if/else 节点。

### 功能五：Queue Mode + 错误处理

支持工作流队列模式，可以处理大规模并发任务。内置重试机制和错误通知。

**为什么好：** 自动化最怕什么？半夜跑崩了没人知道。n8n 的队列模式 + 完善的错误处理，让它从"玩具"变成了"生产级"工具。

## 03 ｜典型场景和避坑

### ✅ 适合谁

**场景一：个人开发者做自动化个人助理**

把 Gmail、日历、Todoist 串起来，做个 AI Agent 帮你过滤邮件、安排日程、自动回复。我见过有人用 n8n 搭了一个"个人 CEO 助理"，每天自动总结邮件、排优先级、写待办清单。

**场景二：小公司做客服自动化**

AI Agent + 飞书机器人 + 知识库。客户在飞书群里提问，Agent 自动检索 FAQ 回答，解决不了的转人工。30 分钟的活变成 30 秒。

**场景三：运营团队做数据 Pipleline**

从 数据库拉数据 → AI 分析生成报告 → 自动发邮件/发飞书消息。一套流程跑下来，周报月报再也不用手动写。

### ❌ 不推荐给谁

**纯业务人员，没有任何技术背景**

n8n 虽然号称 no-code，但你真的啥都不懂的话，填 API Token 这一关就能劝退。Setup 门槛比 Zapier 高——毕竟你要自己部署。建议先玩 n8n Cloud 的免费额度试试水。

**需要复杂分布式工作流的大型企业**

n8n 的单实例模式在大规模下会有性能瓶颈。虽然 Queue Mode 改善了很多，但如果你要处理每天百万级的任务，还是考虑专业的 ETL 工具更稳妥。

## 04 ｜上手教程

### 方式一：Docker 一键部署（推荐）

```bash
docker run -it --rm \
  --name n8n \
  -p 5678:5678 \
  -v ~/.n8n:/home/node/.n8n \
  n8nio/n8n
```

打开 `http://localhost:5678` 即可。

### 方式二：Docker Compose（带持久化）

```yaml
version: '3.8'
services:
  n8n:
    image: n8nio/n8n
    ports:
      - "5678:5678"
    volumes:
      - ~/.n8n:/home/node/.n8n
    environment:
      - N8N_SECURE_COOKIE=false
```

### 方式三：n8n Cloud（最简单）

直接去 [n8n.io](https://n8n.io) 注册，有免费额度，不用部署直接玩。

### 3 分钟创建一个 AI 工作流

1. 打开 n8n，点击 "New Workflow"
2. 拖一个 **Manual Trigger** 节点（手动触发）
3. 拖一个 **AI Agent** 节点，连接到 Trigger
4. 在 AI Agent 配置里：
   - 选择模型：OpenAI / Claude 或其他
   - 粘贴一个 API Key
5. 拖一个 **HTTP Request** 节点，设成你要调用的外部 API
6. 连接 AI Agent → HTTP Request，让 Agent 决定何时调用
7. 点击 "Execute Workflow"，在输入框里问一个问题
8. 看 Agent 自动推理、调用工具、返回结果

整个过程不到 5 分钟。
