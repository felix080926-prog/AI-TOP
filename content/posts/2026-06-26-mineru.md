**大家好，我是雷达君。**

你有没有遇到过这种场景：你手上有一份重要的 PDF——可能是学术论文、产品白皮书、或者一份带复杂表格的行业报告。你想把它喂给大模型做分析，结果发现：

- **公式全乱了**：好好的数学公式变成了一堆乱码
- **表格稀碎**：跨页的表格被拆得七零八落，合并单元格全丢了
- **多栏文本错位**：学术论文那种双栏排版，提取出来成了“左一句右一句”的乱序
- **扫描件直接没法用**：纯图片的 PDF，大模型根本读不懂

你试了市面上各种 PDF 转文本工具，要么收费高得离谱，要么解析精度感人。最后只能手动复制粘贴——一个下午就耗在一份文档上了。

**PDF 的“只读”属性，成了 AI 读取知识的最后一道墙。**

今天雷达君要安利的这个开源项目，就是来拆这道墙的。它由**上海人工智能实验室（上海 AI Lab）** 出品，能把 PDF、DOCX、PPTX、XLSX、图片、网页等复杂文档，精准转换成结构清晰、AI 直接可读的 Markdown 或 JSON 格式。

它就是——**MinerU**。

## 01 ｜核心逻辑：把“不可读”的文档，变成 AI 的“母语”

MinerU 的核心理念非常直接：**文档格式不应该成为 AI 获取知识的障碍。**

PDF 的设计初衷是“跨平台保持排版一致”——它天生就是给人看的，不是给机器读的。PDF 里的文字、图片、表格、公式被“锁”在了一个固定的视觉布局里，机器拿到它就像拿到一张照片：能看到，但读不懂。

MinerU 做的事情，就是把这张“照片”**逆向工程**回结构化的数据。

它基于多模块文档解析策略，通过精细调优的预处理和后处理规则来保证最终结果的准确性。不管是文本型 PDF、图层型 PDF 还是扫描版 PDF，MinerU 都能处理。它内置了 VLM+OCR 双引擎，支持**109 种语言**的 OCR 识别。

**一句话总结：你给 MinerU 一份“给人看的文档”，它还你一份“给 AI 读的数据”。**

## 02 ｜硬核功能盘点

> 🔗 **GitHub 地址**：https://github.com/opendatalab/MinerU

### 1. 全格式覆盖：PDF、Office、图片、网页，一把梭

MinerU 不是只能处理 PDF 的“偏科生”。它原生支持**DOCX、PPTX、XLSX**的解析，也支持图片和网页。

- **公式** → 输出 LaTeX 格式
- **表格** → 输出 HTML 格式，精准还原布局
- **多栏文本** → 按人类阅读顺序重组，自动去除页眉页脚
- **扫描件/手写体** → VLM+OCR 双引擎自动识别
- **跨页表格** → 自动合并，不断裂

你不需要关心文档是什么类型、什么格式、什么语言——丢给 MinerU，它帮你搞定。

### 2. MinerU 2.0：速度提升 6 倍，准确度提升 22%

2025 年 8 月，MinerU 升级到**2.0 版本**，实现了质的飞跃：

- **解析速度提升 6 倍**
- **综合准确度提升 22%**
- **模型参数量仅 0.9B**，但综合评分在同级别开源模型中排名第一，甚至比肩 72B 的主流大模型

> 在最新发布的**3.4 版本**（2026 年 6 月 18 日）中，MinerU 的 OCR 能力再次升级。OCR 模型升级到**PP-OCRv6**，在 OmniDocBench v1.6 上 OCR 准确率提升约**11%**。OCR 处理速度提升约**100%**。用更少的算力，拿更高的精度。

### 3. 生态集成：LangChain、Dify、Claude Code 全打通

MinerU 不是孤岛。它提供了**MCP Server**，可以直接接入 Cursor、Claude Desktop、Windsurf 等 AI 编程工具。同时原生集成**LangChain、LlamaIndex、RAGFlow、Dify、FastGPT**等主流 RAG 和 Agent 框架。

你可以在 Claude Code 里让 AI 直接调用 MinerU 解析 PDF，也可以在 Dify 的工作流里把 MinerU 作为文档预处理节点——**它不是你手动跑的命令行工具，而是你 AI 工作流里的一个“原生组件”。**

### 4. 部署灵活：CPU 也能跑，国产 AI 芯片全适配

MinerU 提供了三种推理后端：

- **pipeline**：快速稳定，无幻觉，CPU 或 GPU 都能跑
- **vlm-engine**：高精度，支持 vLLM/LMDeploy/mlx 生态
- **hybrid-engine**：高精度+原生文本提取，低幻觉

特别值得一提的是，MinerU 已经适配了**超过 10 种国产 AI 芯片**，包括昇腾、寒武纪、燧原、天数智芯、摩尔线程、昆仑芯、沐曦、海光、壁仞、平头哥等。**国产算力+国产文档解析引擎，数据全链路不出国门。**

### 5. 零门槛使用：Web 版、桌面端、API，三选一

MinerU 提供了多种使用方式，从“零代码”到“开发者”全覆盖：

- **Web 版**：浏览器直接使用，无需安装
- **桌面客户端**：下载即用，无需编程
- **API**：一键接入现有系统
- **Python SDK / CLI / REST API / Docker**：开发者全方案

> **现实摩擦提醒**：MinerU 的本地部署对 GPU 资源有一定要求。如果没有 GPU，也可以用 CPU 跑 pipeline 模式——速度会慢一些，但依然可用。不想折腾部署的话，直接上 Web 版或桌面端就行。

## 04 ｜典型场景和避坑

**适合谁用？**

- **构建 RAG 系统、需要从海量 PDF 中提取高质量语料的 AI 工程师**：MinerU 是目前开源方案里精度最高的选择之一
- **需要解析学术论文、技术白皮书、行业报告的科研人员/分析师**：公式、表格、多栏排版，MinerU 全都能精准还原
- **想把企业内部文档（Word、PPT、PDF）统一转化为 AI 可读格式的企业 IT 团队**：全格式覆盖 + 国产芯片适配，数据安全可控
- **在用 Claude Code、Cursor 等 AI 编程工具，需要让 AI 读取 PDF 文档的开发者**：MCP Server 一键接入

**不推荐给谁？**

- **只需要提取纯文本、不关心格式和结构的轻量场景**：用 PyPDF2 或 pdfplumber 可能更轻量
- **完全没有技术背景、也不想用 Web 版/桌面端的用户**：虽然 MinerU 有零门槛方案，但本地部署仍需一定技术基础

## 05 ｜快速上手指南

MinerU 提供了多种使用方式，从“零代码”到“开发者”全覆盖：

### 方式一：Web 版（最推荐零基础用户）

直接访问[https://mineru.net](https://mineru.net)，上传文档即可在线解析，无需安装任何软件。

### 方式二：桌面客户端（推荐非技术用户）

访问官网下载桌面客户端，安装后直接使用，无需编程。

### 方式三：Python 安装（推荐开发者）

```bash
pip install mineru
```

安装完成后，一行命令即可解析文档：

```bash
mineru -p /path/to/your/document.pdf -o /output/dir
```

### 方式四：Docker 部署（推荐生产环境）

```bash
docker run -d -p 8000:8000 opendatalab/mineru:latest
```

### 方式五：MCP Server 接入（推荐 AI 编程工具用户）

在 Claude Code 或 Cursor 中配置 MinerU 的 MCP Server，即可在 AI 对话中直接调用 MinerU 解析文档。

## 06 ｜总结

MinerU 解决的核心问题是什么？

**PDF 的“只读”属性，一直是 AI 读取知识的最后一道墙。**

学术论文、行业报告、产品白皮书、企业内部文档——人类最有价值的知识，大量被锁在 PDF 里。而 AI 能读懂的，是结构化的文本。

MinerU 用一套 VLM+OCR 双引擎 + 精细预处理/后处理规则 + 全格式覆盖的方案，把这堵墙拆了。公式 →LaTeX，表格 →HTML，多栏 → 阅读顺序重组，扫描件 →OCR 识别。输出的是 AI 可以直接消费的 Markdown 和 JSON。

当你的 AI 终于能读懂那些“锁在 PDF 里的知识”时，你会发现：**不是模型不够强，是数据喂得不够好。**
