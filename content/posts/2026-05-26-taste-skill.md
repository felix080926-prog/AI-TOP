大家好，我是雷达君。

你有没有被 AI 写出的话气笑过？

上周我让 ChatGPT 帮我写一封"婉拒合作"的邮件，它洋洋洒洒来了几百字，开头是："感谢您的来信，我们深感荣幸能与贵方探讨合作的可能性……" 每个字都对，但读完整个人都不好了——像极了那种刚毕业的实习生，穿着借来的西装，满口"尊敬的、恳请的、荣幸之至"。

这不是 AI 不聪明，是 AI 没品味。

今天要聊的项目就叫 **taste-skill**，GitHub 上不到一个月攒了 2 万星，做的事情用一句话说就是：**让 AI 学会什么叫"好品味"，别再写那种让人起鸡皮疙瘩的电子废话了。**

---

## 01 ｜核心逻辑

先说一个反常识的事实：现在的 AI 模型其实**能力过剩，但审美为零**。

GPT、Claude、DeepSeek 这些模型，训练数据里塞了几百亿条文字。它们知道怎么写一篇"合格"的文章——有开头、有结尾、有逻辑、有礼貌。问题就在于**太合格了**。

合格到每个人写的 AI prompt 产出的东西都一个味儿：客套、圆滑、面面俱到、毫无个性。就像你们公司那个开会永远说"大家说得都有道理"的产品经理——他不犯错，但你也记不住他。

taste-skill 的本质就是一本 **AI 审美手册**。

它是一个 skill 文件——你可以把它注入到 Cursor、Windsurf、Claude 这些工具的 system prompt 里。它会告诉 AI：

> "别写那些套话。说你真正想说的。省掉'当然''我很乐意'这种废话。有自己的态度。"

它不是教 AI 更多的知识，而是在教 AI **选择**。就像教一个人写文章，不是教他认更多的字，而是教他什么话该说、什么话该删、什么语气讨人喜欢。

这个思路牛不牛？**能力和品味的分离**——模型负责"会不会"，taste-skill 负责"好不好"。

## 02 ｜硬核功能盘点

> 🔗 GitHub 地址：https://github.com/Leonxlnx/taste-skill

**功能一：去 AI 味规则引擎**

taste-skill 最核心的就是一套精心设计的规则集。它不是一个 AI 模型，它是一份高质量的行为指南。每条规则都针对一个具体的"AI 废话"场景：去掉"当然""值得注意的是""不可否认的是"这些 AI 口头禅，砍掉啰嗦的前言后语，直接进入正题。

**功能二：场景化品味配置**

不同场景需要不同的"品位"。写技术文档和写小红书帖子，语气天差地别。taste-skill 按场景提供了多套风格配置：

- **简洁模式**：能用 5 个字说清楚的事绝不用 50 个字
- **专业模式**：去掉"我觉得""可能吧"这类弱化词，直接给结论
- **自然模式**：像真人聊天，该开玩笑开玩笑，该认真认真

**功能三：可扩展的规则体系**

项目本身是开放的，你可以在它的基础上加自己的规则。比如你特别讨厌 AI 用"深刻地""深入地"这类词，直接在规则文件里加一条就行。

## 03 ｜典型场景和避坑

**适合谁用？**

👆 **任何一个每天跟 AI 打交道的人**

- 内容创作者：写公众号、小红书、短视频脚本，AI 初稿太啰嗦，直接用 taste-skill 过滤一遍
- 程序员：用 Cursor/GitHub Copilot 写代码注释、技术文档，一上来 AI 就写一段"该函数旨在……"——快停下，用 taste-skill
- 运营/产品经理：写周报、写需求文档、写 PRD，AI 写的永远像用翻译软件翻出来的
- 团队协作：你和你的同事都用同一套 taste-skill 配置，所有人都能写出风格统一的文档

**不推荐给谁？**

👇 **追求"一键出精品"的人**

taste-skill 不是魔法棒。你得花 10 分钟看看它的规则文件，理解它在做什么，可能还要根据自己的偏好做微调。如果你只想对着 AI 说"帮我写一篇好文章"然后什么都不管，taste-skill 救不了你。

还有一个坑：**过犹不及**。规则的量要适中。如果你加的规则太多太严格，AI 可能会写出一种"刻意酷"的感觉——反而更尴尬。适度放水，留一点 AI 原本的味道，反而更自然。

## 04 ｜上手教程

简单到离谱，5 分钟的事。

**方式一：在 Cursor/Windsurf 中使用**

把 taste-skill 的内容复制到项目的 `.cursorrules` 或 `.windsurfrules` 文件里：

```bash
# 克隆项目
git clone https://github.com/Leonxlnx/taste-skill.git

# 进入目录
cd taste-skill

# 把 taste-skill.md 复制为你的项目规则文件
cp taste-skill.md .cursorrules

# 重新打开 Cursor，AI 就已经有了"好品味"
```

**方式二：在 Claude 或其他通用 AI 中使用**

直接复制 taste-skill.md 的内容，粘贴到对话的 system prompt 里：

```
<system_prompt>
[这里粘贴 taste-skill.md 的全部内容]

请用简洁、直接、有态度的语气回答。不要客套。去掉"当然""值得注意的是""不可否认的是"这类废话。
</system_prompt>
```

**方式三：作为 API 调用时的 system prompt**

```python
import anthropic

with open("taste-skill.md", "r") as f:
    taste_rules = f.read()

response = client.messages.create(
    model="claude-sonnet-4-20250514",
    system=taste_rules,
    messages=[{"role": "user", "content": "写一段产品介绍"}]
)
```

**验证效果最简单的方法：**

在配置前后，让 AI 用同样的 prompt 写同一段话。你会看到配置前的版本和配置后的版本判若两人——一个像在春晚念稿，一个像在跟你喝咖啡聊天。

## 05 ｜总结

taste-skill 做的事情其实很小——它只是一份配置文件。但它解决了一个巨大的问题：**当 AI 的能力已经过剩，最稀缺的反而是审美和品味。**

你不能指望模型自己学会"好品味"。训练数据里既有莎士比亚也有小红书营销号，AI 根本分不清哪个该学哪个该扔。你需要一个人来告诉它——你说出来的话，**应该有辨识度、有态度、有个性**。

taste-skill 就是那个人。

它不会让你写得更快，但它会让你写出来的东西更像"你写的"。在 AI 内容越来越同质化的今天，这可能比任何技术突破都值钱。
