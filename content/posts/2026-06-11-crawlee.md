**大家好，我是雷达君。**

你有没有遇到过这种场景：辛辛苦苦写了一个爬虫脚本，跑了十几页就 IP 被拉黑；刚把 Python 脚本调试好，对方网站改了个版，你的 XPath 全废了；跑了一夜的数据因为某个请求超时，整个任务直接崩溃——第二天到公司发现什么都没抓到。

这不是你的问题。这是现代网站反爬的日常。浏览器指纹检测、速率限制、动态内容渲染、无限滚动加载……现代网页早就不是“下载 HTML 再正则解析”就能搞定的古董了。

市面上能写爬虫的库不少。Puppeteer 会操作浏览器但不会管 URL 队列，Playwright 能处理动态内容但不自带去重和重试机制。你拼了三个库搞定一套流程，结果代理池又不稳定，每天被反爬搞得焦头烂额。

今天雷达君安利的这个开源项目，专治以上所有症状。它不是又一个“脚本玩具”，而是把一家年营收数千万美元的爬虫 SaaS 公司——**Apify**——内部沉淀了多年的工业级爬虫引擎，完整开源给了你。

它就是——**Crawlee**。

## 01 ｜核心逻辑：把你的爬虫脚本，武装成“工业装甲车”

Crawlee 由 Apify 团队开发并开源，是目前 Node.js 生态里最成熟、最完整的 Web 爬取与浏览器自动化框架。Apify 这家公司做爬虫 SaaS 做到了数千万美元年营收，Crawlee 是他们内部每一行代码都被生产环境反复锤炼过的核心引擎。

Crawlee 的核心逻辑是：**一套 API 打通五种爬虫模式，内置反检测、代理管理、请求队列、自动缩放——你需要的一切，开箱即用**。

你写过 Puppeteer 吗？写过 Playwright 吗？每次写个爬虫，你得自己操心 URL 队列的去重、并发数量的控制、请求失败的重试逻辑、代理怎么轮换……这些重复性的“胶水代码”，写一次累一次。Crawlee 把这套脏活全包了。

只要把注意力集中在“从页面里提取什么数据”这一个事情上，剩下的路由、调度、反封、存储、重试，Crawlee 全自动帮你搞定。

## 02 ｜硬核功能盘点

> 🔗 **GitHub 地址**：https://github.com/apify/crawlee

Crawlee 不是靠着“功能多”堆起来的 Star 数。在 2026 年还能脱颖而出，靠的是以下四条环环相扣的硬核能力：

**1. 五种爬虫，一套 API：从静态到动态，一把梭**

Crawlee 不做“一把锤子锤所有钉子”的事，而是针对不同网页类型提供最合适的爬虫，但它们共用一套 API——这意味着你可以从最轻量的方案起步，遇到复杂页面再切换到浏览器引擎，代码结构几乎不用改。五个爬虫器分工如下：

- **CheerioCrawler**：最快、最轻量。用 Cheerio 解析 HTML，纯 HTTP 请求，不跑 JavaScript。适合静态网页、新闻站、目录页。速度堪比火箭，但碰到 React/Vue 就歇菜。
- **JSDOMCrawler**：速度很快，可以创建轻量级 DOM 环境，比真实浏览器省资源。
- **PlaywrightCrawler**：主力选手。用 Playwright 驱动真实浏览器，支持 Chromium、Firefox、WebKit 三大内核。适合 SPA、无限滚动、表单交互等复杂场景。
- **PuppeteerCrawler**：针对 Google 技术栈优化，适合用 Puppeteer 的开发团队。
- **BasicCrawler**：底层基类，自定义一切，适合那些不解析 HTML 的特殊任务。

> 核心思路很简单：用 CheerioCrawler 跑 95%的静态页面，遇到 JS 渲染的页面再自动切换到 PlaywrightCrawler。不需要重写代码，改一行参数就行。

**2. 反检测设计：让你的爬虫看起来像一个“真人”**

这可能是 Crawlee 最核心的硬实力。Apify 团队花了数年时间打磨的反检测能力，默认配置下就能让你的爬虫伪装成真实浏览器，绕过多数的 bot 检测。

Crawlee 反检测机制包含以下王牌组合：

- **TLS 指纹伪装**：模拟 Chrome、Firefox 的真实 TLS 握手特征；
- **Stealth 模式**：补齐 WebGL 签名、Canvas 指纹、字体列表等容易被 bot 检测抓住破绽的浏览器环境信息；
- **动态 headers**：每个请求带上的 headers 跟真实浏览器一模一样，不走样；
- **Session 池 + Cookie 管理**：自动维护会话状态，减少验证码触发频率；
- **代理轮换**：内置 ProxyConfiguration，支持集成主流代理服务商，自动切换出口 IP、剔除失效 IP。

最关键的是，这一切默认开启，零配置即可生效。不需要你自己拼凑一堆 Stealth 插件、fingerprint 库，Crawlee 出厂就内置好了。

**3. 生产级工程能力：企业规模也能扛住**

当你需要爬几百万个页面时，Crawlee 内置的生产级存储引擎就派上用场了：

- **RequestQueue**：URL 队列支持 BFS/DFS 遍历，自动去重，理论规模无上限。
- **Dataset**：结构化数据存储，自动持久化为 JSON，支持导出 CSV/XLSX。
- **KeyValueStore**：存文件、截图或任意数据，适合存中间状态。

除此以外，以下能力更是企业级痛点必选项：

- **自动缩放**：根据 CPU 负载、内存、网络延迟动态调节并发数；
- **指数退避重试**：请求失败后自动重试，避免瞬时风控误判；
- **请求优先级管理**：高价值页面优先处理，低价值页面后置；
- **持久化存储**：开发环境和生产环境用同一套存储机制，无痛迁移；
- **导出格式自由切换**：JSON、CSV、NDJSON，或对接数据库。

**4. AI 爬虫新时代：一句话让 AI 替你写选择器**

Crawlee v3.16 发布了一个改变规则的重磅更新——**StagehandCrawler**。这是一个用自然语言与网页交互的爬虫，告别又脆又容易过期的 CSS 选择器。你不需要费力写精准的 XPath 路径和 CSS 选择器，只需要用英语描述需求：

```typescript
// 传统做法：手写脆弱的选择器
const products = await page.$$eval('.product-card div.price', ...)

// Stagehand做法：AI理解你在说什么
await page.act('点击“加载更多”按钮')
const data = await page.extract('获取所有商品名称和价格', schema)
```

StagehandCrawler 完美继承了标准 Crawlee 的所有能力——请求队列、代理轮换、自动缩放、会话管理、浏览器指纹伪装——一套都不落下。

> _现实摩擦提醒_：StagehandCrawler 依赖第三方 AI 模型 API（如 OpenAI），因此会产生额外成本，且响应速度相对传统的 PlaywrightCrawler 较慢。如果目标网站的 HTML 结构稳定且不频繁变样，传统爬虫仍是更高效、零成本的方案。

## 03 ｜与 Crawl4AI 的核心区别

2026 年开源爬虫圈里，最引人注目的两股力量当属 Crawlee 和 Crawl4AI。但它们的定位完全不同，甚至可以说是在爬取链路的**两端**。

| 维度           | **Crawlee**                                       | **Crawl4AI**                                   |
| :------------- | :------------------------------------------------ | :--------------------------------------------- |
| **核心定位**   | 工业级浏览器自动化与爬虫框架                      | LLM 友好型网页内容提取工具                     |
| **技术栈**     | Node.js / TypeScript                              | Python                                         |
| **设计哲学**   | 爬取任意网页的“脚手架”，开发者掌控一切            | 专注于把网页内容转化成 LLM 最爱吃的格式        |
| **反检测能力** | 出厂自带 TLS 指纹、代理池、Stealth 模式           | 自带反爬能力偏弱，集成较多依赖第三方           |
| **输出格式**   | 结构化数据（JSON/CSV）、HTML 文件                 | **原生 Markdown**、结构化数据                  |
| **AI 集成**    | StagehandCrawler（自然语言操作元素）              | 专为 RAG 和 LLM 数据管道设计的 html2markdown   |
| **最佳场景**   | 需要大规模、高频率、强稳定性数据采集              | 一次性采集数据后做 RAG、喂 LLM 分析内容        |
| **典型用户**   | 数据工程师、DevOps、需要规模化数据采集的 SRE 团队 | 做 RAG 系统、Agent 开发、大模型微调的个人/团队 |

概括一下：**Crawlee 是“怎么拿到数据”的专家，Crawl4AI 是“怎么让 AI 读懂数据”的专家。**

如果你的目标是爬取几十个竞品网站的商品信息、监测全网舆情变化、抓取 SERP 结果，Crawlee 凭借工业级的稳定性、反检测能力和规模化调度能力，是更合适的选择。如果你的目标是一次性获取网页后给 LLM 做分析或 RAG，Crawl4AI 在 Markdown 输出和 LLM 友好性上有独特优势。更高阶的打法是把 Crawl4AI 嵌入 Crawlee 的 Pipeline——Crawlee 拿到 HTML 后用 Crawl4AI 转成 Markdown，两者双剑合璧。

## 04 ｜典型场景和避坑

**最适合谁用？**

- **爬虫频率高、数据量大、容不得出错的商业化项目**：电商价格监控、房地产市场数据、招聘信息聚合——爬虫挂了，业务就停了。
- **需要爬取动态 JS 渲染页面、绕过 Cloudflare 等反爬措施的团队**：Crawlee 的 PlaywrightCrawler + 默认反检测配置能扛过绝大多数字段。
- **企业内做数据采集平台或数据基础设施的研发团队**：Crawlee 为企业级团队准备了一整套可扩展的请求队列、持久化存储和 API。

**不太适合谁？**

- **只需要一次性抓取几个网页做数据分析的非技术研究人员**：用 Python 写几行 requests 加 BeautifulSoup 可能更快。
- **完全没有 Node.js 经验、也不想接触 JavaScript 的团队**：Crawlee 专注于 JS/TS 生态，虽然官方近年已推出 Python 版，但 Python 版在生态成熟度和功能完整性上仍处于成长期。
- **只想把网页内容转成 Markdown 喂给 LLM**：这个问题用 Crawl4AI 会比 Crawlee 更顺手、代码更少。

## 05 ｜快速上手指南

Crawlee 的手感极其流畅。如果你写过一个 Puppeteer 脚本，Crawlee 会让你产生“这就是我梦寐以求的脚手架”的亲切感。

**环境要求**：Node.js 16+

**第 1 步：CLI 一键创建**

```bash
npx crawlee create my-crawler
# CLI引导选择爬虫类型（CheerioCrawler / PlaywrightCrawler）
cd my-crawler && npm start
```

CLI 自动完成目录创建、依赖安装、示例代码的全部配置工作。

**第 2 步：手动写一个爬虫（展示 Crawlee 的极简性）**

```typescript
import { PlaywrightCrawler, Dataset } from 'crawlee';

const crawler = new PlaywrightCrawler({
  async requestHandler({ request, page, enqueueLinks, log }) {
    const title = await page.title();
    log.info(`Title of ${request.loadedUrl} is '${title}'`);
    // 数据自动持久化到 ./storage/datasets/default
    await Dataset.pushData({ title, url: request.loadedUrl });
    // 提取页面所有链接，自动加入爬取队列
    await enqueueLinks();
  },
});

await crawler.run(['https://crawlee.dev']);
```

这段脚本完成了：

- 打开目标网站 → 提取标题和 URL → 自动存入本地 JSON
- 递归提取页面上的所有链接 → 自动加入队列深度爬取
- 全程 Crawlee 替你管理反爬策略、队列、并发和持久化

**第 3 步：进阶配置——代理池和浏览器指纹**

```typescript
const crawler = new PlaywrightCrawler({
  proxyConfiguration: await ProxyConfiguration.create({
    proxyUrls: ['http://proxy1', 'http://proxy2'],
    // 自动轮换、自动剔除失效代理
  }),
  launchContext: {
    // 自动补全浏览器指纹、调整headers
    stealth: true,
  },
});
```

如果你不想在本地跑，Crawlee 原生支持部署到 Apify 云平台，配合 Docker 一键上线。

## 06 ｜总结

一个真正意义的爬虫，不应该只是“能跑起来就行”。

现代 Web 越来越复杂——动态渲染、反爬指纹、分布式限流、会话审计……每一次技术升级都在提高数据采集的成本和门槛。Crawlee 带给爬虫开发者的，是一套经过 Apify 在真实战场淬炼的企业级“装备箱”：五种爬虫引擎覆盖所有场景，出厂内置的反检测伪装术让你顺畅穿过大多数反爬墙，自动缩放、队列管理和持久化存储让你从单机脚本平稳过渡到百万级页面的规模化任务。

它不是普通的爬虫库，而是你写爬虫时的“钢铁侠战甲”。

Apache 2.0 开源，超 23k GitHub stars，由 Apify 团队主力维护，全球数万企业生产环境验证。
