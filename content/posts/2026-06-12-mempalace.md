**大家好，我是雷达君。**

你见过能写代码、能画图、能做分析报告，但唯独记不住“你上周跟它说过什么”的 AI 吗？

和 Claude Code 或 Cursor 的 Agent 聊上几轮，你就会发现一个致命的短板：**它根本不记得 10 分钟前自己说过什么，更别提前几天讨论过哪个决策。**

每到会话开始，你都得重新向它解释一遍项目的背景、你已经尝试过的方案、之前被否决的方向——AI 一直在输出，但它的长期记忆仿佛存不进硬盘的 SSD，随时断电随时清零。

市面上已有不少“AI 记忆”工具，比如 Mem0 和 Zep，但它们都有一个致命的缺陷：**它们用 AI 来决定什么值得记住**。模型会先做一轮“摘要”或“智能提取”，然后存下它自己筛选出来的信息——那些恰恰被 AI 认为是“不重要”的细节、微妙但关键的前提条件、后来被推翻的初始假设……恰恰是最具价值的信息。

今天雷达君要安利的这个开源项目，彻底否定了这条技术路线。

它的开发者之一，是**《生化危机》的女主角米拉·乔沃维奇**——没错，就是那个在电影里打丧尸的米拉。她找来程序员朋友本·西格曼，两人用 Anthropic 的 AI 编程工具花了几个月时间，把这个“记忆宫殿”写了出来。

它就是——**MemPalace**。

## 01 ｜核心逻辑：不总结、不摘要、不提取——存所有，让搜索帮你找

绝大多数 AI 记忆工具走的路线都是“智能筛选”——用 AI 读完对话，生成一段摘要，压缩成几条“关键信息”存起来。这个逻辑看似聪明，实则脆弱。因为你永远不知道 AI 自作主张丢弃的那条信息，是不是你明天决策的关键依据。

MemPalace 做了完全相反的选择：**不提炼、不压缩、不决定什么“值得记住”。**

它的做法极其简单粗暴：**存原样。**你把聊天记录、代码对话、会议纪要全都喂给它，它把每条信息原封不动地存进抽屉。每次 AI 需要“回忆”时，它做语义搜索，从原始内容里精准取回相关片段。——没有任何中间环节的信息损耗。

这是一种“先记录下来，需要时再精准找到”的哲学。

MemPalace 把这种存储方式包装成了一个特别直观的隐喻——**“记忆宫殿”**。

灵感源自两千多年前的古希腊记忆术。在古代，演说家把要记忆的事物“摆放”在宫殿的不同房间里，走过一个房间就“翻开”一段记忆。MemPalace 把同样的原理用在了 AI 上——你的对话被组织成**翼（为人和项目）** → **厅（记忆类型）** → **房间（具体想法）** → **抽屉（原始内容）**，四层结构逐级细分。

**你的“宫殿”长什么样，由你自己定。**

> 相比之下，之前的记忆类项目如 Supermemory 走的是“统一记忆层+API+RESTful”路线，需要配合外部向量存储和云端同步；Mem0 走的是轻量级记忆 API 路线。而 MemPalace 的思路是——AI 自己决定存什么不存什么？不，AI 连“决定”的权力都不要。**所有的信息全部原样保留，一个字符都不丢。**
>
> 两种方案没有对错，取舍完全不同。一套方案牺牲存储空间换取信息的绝对完整，另一套方案节省成本但丢失细节。MemPalace 选择了前者。

## 02 ｜硬核功能盘点

> 🔗 **GitHub 地址**：https://github.com/mempalace/mempalace

**1. 基准测试碾压：96.6%召回率，不依赖任何 LLM**

在 AI 记忆领域，最权威的评测数据集叫**LongMemEval**，包含 500 个长跨度情景问答。MemPalace 在原生语义搜索模式下直接跑出了**96.6% R@5**的召回率——这个数字完全不依赖任何 LLM，全程在本地运行，**零 API 调用、零云端费用**。加一层混合增强后，召回率达到 98.4%；再加上 LLM 重排，可以达到 99%以上。

在其他权威基准测试上也全面领先——LoCoMo 长会话检索 88.9%、ConvoMem 跨类别平均召回 92.9%、MemBench 八类数据 80.3%，各项指标名列前茅。

> 项目团队在官方文档中做了一个很严谨的动作——他们拒绝横向对比 Mem0、Zep 等其他项目，因为那些项目发布的是“端到端问答准确率”，而 MemPalace 发布的是“原始召回率”，两个指标的性质完全不同，并排对比不诚实。这种严谨、不吹水的工程态度，在开源社区里极其罕见。

**2. 三层架构 + AAAK 压缩方言 + 时态知识图谱**

MemPalace 的技术深度不止于“存原样”。它包括一套完整的内存管理体系：

- **四层记忆堆栈**：从会话层到永久层，逐级沉淀。“金鱼脑”的问题，本质是模型只在会话层做了短期记忆，而 MemPalace 帮你在系统层做了分级固化。
- **AAAK 压缩方言**：一套专门针对“记忆冲突”和“过时信息”的标记语法。当某个信息已经被新对话覆盖、失效或推翻时，AAAK 方言允许 AI 代理在记录时用特殊标记注明“这条信息已过时”。这带来了传统向量数据库完全不具备的能力——**对记忆状态做标记，让 AI 在检索时可以过滤掉“已经错了”的信息**。
- **时态实体关系图**：不是 RAG，而是带有**时间戳和有效期窗口**的知识图谱。信息不仅是“被记录”，而且是“被编排”。你问“上周的架构决策”，它会去查“有效期窗口包含上周”的那条记录。你问“有哪些决定后来被推翻了”，它会把 AAAK 标记为“已被覆盖”的信息挑出来给你看。**它记住的不只是“是什么”，还有“什么时候有效”。**

**3. 29 个 MCP 工具 + 自动保存钩子，全平台接入**

MemPalace 原生支持**Model Context Protocol（MCP）**，提供 29 个标准化的记忆读写工具，涵盖宫殿读/写、知识图谱操作、跨翼导航、Agent 日记等功能。

它给 Claude Code、Codex CLI 和 Cursor IDE 内置了**自动保存钩子**——会话定期自动保存，上下文压缩前自动快照。你再也不用担心聊了几个小时的对话，因为意外退出或 CLI 卡死而全部丢失。

> Claude Code 的会话在 30 天后会过期，如果没有配置自动保存钩子，你将失去全部对话记录。MemPalace 专门提供了**Claude Code 会话保留检查清单**，帮你把现有的 JSONL 对话备份并一键回填到记忆宫殿中。

**4. 完全本地运行，零 API 费用，数据主权归你**

**这是 MemPalace 与市面上所有闭源记忆工具的又一核心区别。**

MemPalace 不要求任何 API Key，不依赖任何云端服务。整个系统在本地运行，数据存留在你自己的硬盘上。

> 它支持 ChromaDB、SQLite、Qdrant、pgvector 等多种后端可插拔存储引擎。你想把数据存本地还是托管到私有的 Postgres 实例，都随你。

即使你要使用 DeepSeek、Ollama 等本地模型，MemPalace 的向量搜索也是纯本地的。不存在“每调一次记忆就烧几十 K Token”的情况。

**5. 多语言+Agent 日记+多翼架构**

MemPalace 内置的多语言嵌入模型，原生支持中英文及 100+语言。不同项目、不同的人、不同的代理，各自拥有独立的“翼”，共享同一个记忆基础设施但逻辑完全隔离，不会相互污染。

> 每个专业 Agent 在宫殿里都有自己专属的翼和日记，运行时通过`mempalace_list_agents`可发现，不污染系统提示词。

> 如果你用 OpenClaw、Codex 或其他 MCP 兼容工具，也能无缝接入 MemPalace。

## 03 ｜快速上手指南（5 分钟见效）

MemPalace 的安装和上手极其简单，完全本地运行，零 API 费用。

**第 1 步：安装 CLI**

用 uv 或 pipx 在隔离环境中安装，避免依赖冲突：

```bash
# 推荐方式：uv tool install（Mac/Linux）
uv tool install mempalace

# 或使用pipx
pipx install mempalace
```

不想安装 Python 环境？Docker 也能跑：

```bash
docker build -t mempalace .
docker run -i --rm -v mempalace-data:/data mempalace
```

**第 2 步：初始化你的“记忆宫殿”**

```bash
mempalace init ~/projects/myapp
```

初始化过程会引导你选择嵌入模型（推荐 embeddinggemma-300m 多语言模型）。

**第 3 步：往宫殿里存内容**

```bash
# 挖掘项目文件
mempalace mine ~/projects/myapp

# 挖掘Claude Code对话记录
mempalace mine ~/.claude/projects/ --mode convos
```

**第 4 步：连接你的 AI 代理**

在 Claude Code 的 MCP 配置中加入 MemPalace 服务器：

```json
{
  "mcpServers": {
    "mempalace": {
      "command": "docker",
      "args": ["run", "-i", "--rm", "-v", "mempalace-data:/data", "mempalace"]
    }
  }
}
```

配置完成重启 Claude Code 后，你的 AI 代理会自动调用 MemPalace 的记忆工具。

**第 5 步：为 Claude Code 配置自动保存钩子**

按照官方**Claude Code 会话保留检查清单**配置自动保存钩子：备份现有 JSONL 对话，用`mempalace mine`命令一键回填历史会话，并设置定时保存任务。

> 这一步是确保你过去辛苦积累的所有对话内容“永不丢失”的关键。早期版本的用户可能没意识到，Claude Code 的会话在 30 天后会自动过期——如果没有自动保存钩子，你根本拿不回来。MemPalace 的钩子机制正好补上了这个致命短板。

## 04 ｜典型场景和避坑

**极适合谁用？**

- **Claude Code 重度用户**：你过去 10 周、20 周的所有技术和项目的决策历史、被否定的方案、讨论过的架构设计——完整保留，随时回顾。
- **对数据主权极度敏感的企业/团队**：所有记忆数据停留在本地硬盘，没有第三方能接触到。
- **需要 AI 越用越“懂”你的开发者**：会话越多，语义搜索越精准，你和 AI 之间的默契感指数级上升。

**不太适合谁？**

- **只做单次任务、不在乎上下文保留的轻量用户**：用 MemPalace 属于杀鸡用牛刀。
- **想全云端托管、不想碰任何本地部署流程的团队**：MemPalace 的本质就是本地优先，如果你追求零运维的 SaaS 体验，它和你的场景存在天然错位。

## 06 ｜总结

MemPalace 解决的核心问题，不是 AI 能不能记住你，而是 AI 愿意用什么方式来记住你。

今天的大多数记忆工具，本质上还是“模型说了算”。AI 读完你的话，决定哪句“值得”摘出来存，哪句“不值得”丢进垃圾桶。在 AI 的权重计算里，“关键信息”和“说话时的微妙语气”可能被算成一个不同的向量。你永远不知道被抛弃的细节里，是否藏着将来不可或缺的珍贵洞察。

MemPalace 的哲学正好相反——**AI 不能替你决定什么值得记。一切信息都有价值，全部留存。你真正需要的是搜索和检索能力。**

这不是一次技术方案的优化，而是一次对“AI 自主权”的重新定义。它把 AI 对信息的“决策权”拿回到你手里——不是 AI 筛选了信息，是你用搜索精准地找回了信息。

GitHub 55K+ Star 的背后是越来越多开发者、工程师和 AI 深度用户正在觉醒的意识：我的数据我做主，**AI 只负责存储和检索，绝不负责“遗忘”**。
