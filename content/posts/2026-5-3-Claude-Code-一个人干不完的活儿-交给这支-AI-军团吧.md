大家好，我是雷达君。

你遇到过这种情况吗：让 Claude Code 做一个全栈功能，它写得还行。但你说“加个单元测试”，它可能顺手把业务代码也改了。你说“同时做安全审计”，它只能说“稍后提醒我”。

问题不在 Claude Code 不够强，而是**软件工程本来就不是一个人的战场**。

如果，你的 Claude 旁边同时站着五个专门角色呢？一个拆需求、一个写代码、一个跑测试、一个审安全、一个管上下文——它们各自独立，共享信息，同时开工。

这个场景，已经在 Ruflo 里变成现实了。

今天雷达君安利的，就是把 Claude Code 从“单兵”变成“军团”的开源项目。

它就是——**Ruflo**。

## 01 ｜核心逻辑

Ruflo 做的事，叫**群智能编排**。

打个你一听就懂的比方：你开装修公司。

传统模式：请一个全能大师傅，水电木工油漆都会。但一次只能干一个环节，墙没砌好不能走电路，电路走完才能叫木工。装修两个月，换师傅还得重新磨合同。

Ruflo 模式：你不是请一个人，是组一支工程队。工头拆图纸，几个水电工、木工、油漆工同时推进，共享同一套图纸。周期大幅压缩，问题精准定位。

Ruflo 没发明新工种，它只是把 Claude Code 这个“全能大师傅”变成了一张协作网。

**一句话：不是让 AI 更聪明，而是让 AI 能真正高效“干活”。**

## 02 ｜硬核功能盘点

Ruflo 的核心，是一套完整的**群体智能调度体系**。

🔗 **GitHub 地址**：https://github.com/ruvnet/ruflo

**1. Swarm：从一个人到一个军团**

默认搭载 60+ 预配置专业工种：架构师、程序员、测试、安全、DevOps……你不是雇一个 Claude，而是部署了一支 AI 团队。规划者拆任务、执行者写代码、校验者审查、工具 Agent 调 API。分工层层递进，你只负责验收结果。

**2. 75% API 成本直降**

Ruflo 内置三层模型智能路由：简单任务交轻量模型，中等任务交中档模型，只有真正高难度的核心推理才动用“主力选手”。实测在 SWE-bench 上达到 84.8% 解决率，成本降低 75%——钱花在刀刃上。

**3. 自学习与自优化**

所有智能体共享持久记忆层。今天调好的一个复杂任务流程，下次不用再重新教。用得越久，它越懂你的代码库和工作习惯。

**4. 企业级容错 + 内置向量数据库**

即使某个智能体抽风或 API 节点挂了，系统也不会垮（Raft + 拜占庭容错）。原生带 RuVector 向量数据库，RAG 能力直接嵌入执行链路，不需要外挂 Pinecone。

> **现实摩擦**：大规模调度数十个智能体时，关键设计的追溯链路可能变长。建议对超过 10 个子模块的工程，在关键阶段人工复核一次顶层任务分配。

## 03 ｜对比一下

和之前推荐过的 Superpowers、QwenPaw 比，Ruflo 的定位完全不同。简单说：

- **Superpowers**：给你一个人套上规范（需求对齐 → TDD → 代码审查）。
- **QwenPaw**：给你一个独立员工，在钉钉/飞书里自己跑任务。
- **Ruflo**：给你一个 AI 部门，60 多个工种并行协作，你只管验收。

如果你的项目已经大到一个人写不过来，Ruflo 就是你需要的下一级武器。

## 04 ｜典型场景 & 避坑

**适合谁**：

- 用 Claude Code 做全栈但卡在串行环节的团队
- 追求 AI 工程化效能上限的高级开发者
- 需要 AI 长时间自主运行复杂任务的极客

**不推荐给**：

- 只写一次性脚本的个人开发者（大炮打蚊子）
- 对 AI 工程化没需求、想要“一秒出活”的人
- 多语言混编且测试覆盖不高的老旧项目（可能会拆出过多默认重构任务）

## 05 ｜快速上手

```bash
npm install -g ruflo
npx ruflo@latest init --wizard
# 按提示配置 API Key，然后验证
npx claude-flow swarm-status
```

（Ruflo 曾用名 Claude Flow，两者兼容）

## 06 ｜总结

Superpowers 管流程对不对，QwenPaw 管能不能自主执行，Ruflo 管能不能一群 AI 一起高效干活。

把一个 Claude Code 变成一支军团，还能自学习、自优化、跨机器协作。

如果你的单个 AI 已经撑不住复杂工程了，是时候给它上“军团化体制”。
