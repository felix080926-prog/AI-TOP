**标题：官方都认可的 Claude Code 最强插件，把 AI 变成一整个工程团队**

**大家好，我是雷达君。**

你有没有过这种感觉——每次开新项目，你都在做同一件事：手写 CLUDE.md、配 Agent、写 Hook、调 MCP，光配置环境就花掉一两个小时。更扎心的是，你明知道 Claude Code 能做到自动审查代码、驱动 TDD、并行调度子任务——但因为自己没配好，这些能力你基本用不上。

**工具本身很强，但配置它的成本快赶上写代码本身了。**

还有更烦人的：每次和 AI 好不容易把一个技术方案聊透，session 一关，第二天它又失忆了，忘得一干二净。

今天雷达君要安利的这个开源项目，直接把这些问题一把全解决了。它不是一个“增强插件”，而是一套经过 10 个月高强度生产验证的 **AI Agent 调度性能优化系统**，也是 Claude Code 官方生态中公认的“最强双雄”之一。

它就是——**ECC**。

## 01 ｜核心逻辑

ECC 的全称是 **Everything Claude Code**，由开发者 @affaan-m 从 2024 年开始维护。但它的野心远不止“配置合集”。

它的思路很极端：**把你面对的同一个 Claude Code，变成一个分工明确的工程团队。**

以前你是一个人在跟一个万能 AI 对话。你给它一个需求，它直接跳到代码。它“什么都会”，但也意味着“不专业”——没有规划意识、不会主动架构、不会强制 TDD、也不会自动审查。

ECC 的答案是：**给 Claude 装上一套 Agent 分身术。**

你不再直接命令它写代码。而是先“点名”一个专业 Agent，让它干活：**planner** 先做规划，**architect** 评估架构，**tdd-guide** 驱动测试，**code-reviewer** 审查质量，**security-reviewer** 扫描漏洞。

ECC 靠 10 个月高强度使用和黑客松冠军实战数据验证才长成今天的样子。

**一句话总结：不是“你的 AI 变强了”，而是“你的 AI 分裂成了一个军”。**

## 02 ｜硬核功能盘点

> 🔗 **GitHub 地址**：https://github.com/affaan-m/ECC

**1. 28+ 专业子 Agent：给 AI 戴上“专家头衔”**

这是 ECC 最硬核的能力。它预置了大量专业化子 Agent，每个都有明确的职责边界：

- **planner**：写代码前生成实施计划，按步骤拆解需求。
- **architect**：负责系统设计决策和技术选型。
- **tdd-guide**：严格按 RED→GREEN→REFACTOR 驱动开发。
- **code-reviewer**：自动审查代码质量和安全性。
- **security-reviewer**：专门扫描安全漏洞。
- **build-error-resolver**：构建失败时自动诊断和修复。
- **e2e-runner**：自动运行 Playwright 端到端测试。
- **refactor-cleaner**：清理死代码，优化项目结构。

ECC 内置 8 种语言专属审查 Agent：TypeScript、Python、Go、Java、Kotlin、Swift、Rust、PHP，各自涵盖对应生态的代码规范和安全模板。

> **现实摩擦提醒**：ECC 不是自动驾驶系统。你得学会“点名”。输入“使用 planner agent，帮我设计一个登录系统”，AI 输出一份完整的开发计划，而不是直接甩代码。如果你只是说“帮我写个登录功能”，它还是会变回普通的聊天模式。**不要把你指挥 AI 的方式，还停留在‘万能聊’的阶段。**

**2. 156+ 工作流 Skills：手把手教 AI 干活的说明书**

如果 Agent 是 ECC 的“员工”，Skills 就是每个员工手上的“操作手册”。它提供一套完整的高级工作流定义，覆盖从 TDD、安全审查、代码审查、前后端设计模式到持续学习、内容创作等全场景技能。

**3. 指令自动关联 Agent：slash 命令链**

当你使用 `/plan` 命令，系统会自动调用对应的 **planner agent** 并加载相关 Skills 协同工作。其他命令如 `/tdd`、`/code-review`、`/e2e`、`/build-fix` 同样遵循这种“命令 →Agent→ 技能库”的递进关联逻辑。你不需要记住每个 Agent 的全名。

**4. 记忆持久化：让 AI 从“金鱼”变“大象”**

这是 ECC 解决“AI 失忆症”的核心模块。Claude Code 原生在 session 结束后会丢失所有上下文。

ECC 的持续学习系统在每次 session 结束后自动提炼关键模式、决策逻辑、技术债务，写入可复用的记忆文件。下次会话自动加载。几次长对话反复喂饱 ECC 后，它处理类似问题就不需要从头从头解释。

**5. AgentShield 安全审计：三重并行硬核查漏**

AI 写的代码能用，但能用不等于安全。ECC 集成了 AgentShield 安全审计工具，通过 102 条安全规则和 1282 个测试用例扫描敏感信息泄露、权限配置误用、hook 注入风险等漏洞。

更硬核的是深度红队/蓝队审计模式：用三个 Claude Opus 4.6 并行模拟攻击者和防御者，通过对抗性推理全面评估攻击面。大部分传统扫描工具根本做不到这种级别的对抗性测试。

```bash
# 快速扫描
npx ecc-agentshield scan
# 深度审计（三个Opus并行对抗）
npx ecc-agentshield scan --opus --stream
```

**6. 三种配置预设 + MCP 统一管理中心**

ECC 内置三种全局配置模式：**default** 日常开发、**strict** 生产级强制规范（100% 测试覆盖率 + 类型安全 + 向后兼容）、**creative** 原型头脑风暴（允许快速实验和技术债务）。

MCP 管理中心统一维护 Supabase、Exa、Context7 等多达 14 种以上 MCP 服务的配置模板，避免重复配置和版本冲突。

**7. 跨 Harness 兼容 + ECC 2.0 Rust 控制层**

ECC 全面支持 **Claude Code、Codex、Cursor、OpenCode、Gemini、Zed、GitHub Copilot** 等主流 AI 编程环境。

v2.0.0-rc.1 引入了 ECC 2.0 alpha，新增 Rust 控制层提供 `start`、`status`、`resume`、`daemon` 等 CLI 守护命令，以及 brand-voice、social-graph-ranker、google-workspace-ops 等运营型 skill 扩展。

## 04 ｜典型场景和避坑

**适合谁用？**

- **使用 Claude Code/Cursor 但对产出质量不稳定的开发者**：ECC 的多 Agent 分工将出错的概率降到最低。
- **开发大型项目、需要架构设计和代码审查但人力不足的个人/团队**：AI 承担架构师和审查员角色，释放团队专注在业务逻辑。
- **想系统学习 AI 驱动工程最佳实践的人**：ECC 包含了完整的工程方法论和经过验证的实战模板。

**不推荐给谁？**

- **只想快速跑一个脚本、不在乎流程的小功能开发**：点将的交互成本可能会超过你的预期收益。
- **讨厌命令行和配置文件的技术小白**：ECC 的核心价值藏在深度配置和 Agent 交互里，单纯对话式使用无法发挥其威力。
- **使用非主流技术栈的冷门项目**：如果项目涉及小众语言或框架，ECC 预置的 Agent 和 Skills 可能覆盖度不足。查看 ECC 仓库的 language review agents 列表再决定是否值得引入。

## 05 ｜快速上手指南

**方式一：插件市场安装（最推荐 Claude Code 用户）**

```bash
# 在 Claude Code 中
/plugin marketplace add affaan-m/ECC
/plugin install ecc@ecc
```

⚠️ **重要**：Claude Code 插件无法自动分发 rules。安装完插件后，仍需根据需要手动安装 rules。**若跳过此步，ECC 的功能将严重受限，AI 不会自动遵守 TDD、安全审查等规范。**

**方式二：选择性安装 Rules（安装插件的必接步骤）**

```bash
git clone https://github.com/affaan-m/ECC.git
cd ECC
# 只安装你需要的语言（eg：typescript python）
./install.sh typescript python
```

或使用 npm 脚本：

```bash
npm install
./install.sh
```

**验证安装成功**：在 Claude Code 中输入 `/plan`，系统应当自动触发 planner agent 并输出结构化的任务拆解 Plan，而不是直接生成代码。

## 06 ｜总结

回过头来看 ECC，它不是“AI 写代码的加速器”，而是“AI 做软件工程的方法论”。你为每个项目配置的那份 `AGENTS.md`，本质上是一套明确的开发流程、职责边界和编码规范。

ECC 不是你写代码的工具，而是你管理 AI 团队的操作系统。28 个专业 Agent 可以随时调遣，156 个操作流程预装在体系里，安全审计、记忆持久化、全渠道兼容——从“跟 AI 聊天”到“让 AI 团队干活”，ECC 提供了现成的工程中台。

一切 Agent 调度、TDD 驱动、代码审查、持续学习，都围绕一个目标展开——让你和 AI 的关系，从零散的雇佣劳动变成稳定的长期协作。它是你能亲手调度的多 Agent 团队，帮你把 Claude Code 从一个“不太听话的天才程序员”改造为“能协作、守纪律、记得住的工程团队”。
