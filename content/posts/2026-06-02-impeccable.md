**大家好，我是雷达君。**

相信有不少朋友使用 AI 做过网站、应用等设计，但新交出来的界面，和上个月用别的 AI 工具搭的几乎一模一样。又是 Inter 字体，又是紫色到蓝色的渐变，又是卡片套卡片，灰色的文字趴在一堆花里胡哨的背景色上。

你盯着屏幕，技术逻辑无可挑剔，但视觉上总透着一种说不出的“AI 味儿”。

功能没毛病，就是看起来像个实习生一个下午赶出来的速写。

程序员不会说设计语言，AI 的训练集又充斥着千篇一律的 SaaS 模板，双方的鸿沟最终落成了屏幕上一眼就能被认出的“AI 痕迹”——这正是当今生成式 AI 前端领域最普遍的痛点。

今天雷达君安利的这个开源项目，专治这个病。它由前 Google Chrome DevTools 产品负责人、jQuery UI 的缔造者 Paul Bakaus 亲手打造，给 AI 武装上一整套顶尖设计师的“思维框架”。

用一句话概括：它不仅仅是让 AI 写出更漂亮的界面，而是让**AI 像一个真正懂设计的专家那样去思考**。

它就是——**Impeccable**。

## 01 ｜核心逻辑

Impeccable 的核心理念很简单：**好设计并非天降神谕，而是有一整套精确的词汇与规则。**

当你通过 Claude Code 说“做一个登录页”时，模型没有设计语境，自然只能从训练数据里调用它的万能公式——Inter 字体、紫色渐变、卡片套卡片。

Impeccable 解决的不是补代码，而是为 AI 架设一座体系化的桥梁，灌输一套完整的**设计元语言**。开发者不需要成为设计大师，只需掌握像 `/polish`、`/critique`、`/audit` 这类简短、专业的口令，就能瞬间调用 Impeccable 内置的海量设计智库。

这就像你给一位技术顶尖但毫无美感的程序员配备了一个沉浸式的设计导师。你不用指手画脚，只需说一句“请优化一下”，AI 就会自动理解什么是布局节奏、配色对比、动感动线，从而把一堆中规中矩的组件变成具有专业质感的界面。

## 02 ｜硬核功能盘点

> 🔗 **GitHub 地址**：https://github.com/pbakaus/impeccable

**1. 7 大设计域参考文件：把设计学院的教材，喂给了 AI**

Impeccable 最扎实的底层，是 7 份领域驱动的参考文件，犹如一部浓缩版的“设计圣经”。AI 在回答前会强制加载并学习这些专业知识：

- **Typography（字体排印）** ：字体系统、配对规则、比例缩放、OpenType 特性，甚至解决了 FOUT（无样式字体闪烁）并教会 AI 判断何时不该使用`clamp()`。
- **Color & Contrast（色彩与对比度）** ：利用 OKLCH 色彩空间选色技巧、着色的中性色体系、深色模式适配，以及 WCAG 无障碍标准。
- **Spatial（空间布局）** ：8px 网格体系、间距系统、视觉层级、流体与容器查询设计。
- **Motion（动效设计）** ：缓动曲线、交错动画、偏好减弱动效处理。
- **Interaction（交互设计）** ：表单、聚焦状态、加载模式等细节打磨。
- **Responsive（响应式设计）** ：移动优先策略、视口单位、分网格断点。
- **UX Writing（文案体验）** ：按钮文案、报错提示、空状态信息。

**2. 23 个专业控制指令：完全掌控 AI 的审美**

Impeccable 内置了 23 条直接可用的斜杠命令，构成了一套完整的“对话式设计操控台”：

- `/shape`：边规划边写代码，在动手前先理清 UX/UI 逻辑。
- `/craft`：完整的构思-构建-视觉迭代闭环流程。
- `/critique`：对现有设计进行 UX 审查，评估视觉层次、信息架构和情感表达。
- `/audit`：硬核技术扫描——可访问性、性能表现、响应式兼容性，一次性拉网排查。
- `/polish`：最终交付前的精细打磨、对齐微调、细节优化。
- `/distill`：提炼核心设计语言，剥离冗余堆砌。
- `/animate`：增加目的驱动的动效，而非炫耀性装饰。
- `/colorize`：在单色系中引入策略性色彩。
- `/overdrive`：突破常规设计边界，注入炫技创意。

你将告别繁琐的“prompt engineering”，只需一条精确的口令，就能发动 AI 的整段设计思维链，彻底掌控视觉输出。

**3. 27 条反模式规则 + 12 条 LLM 审查：明确告诉 AI，什么绝对不能做**

Impeccable 最具特色的能力，是用一套包含 27 条确定性规则的反模式集，像代码审查工具一样扫描前端产出物，并结合 12 条 LLM 审查机制做深层解读。

它直接禁止 AI 做这些事：

- 闭眼默认 Inter 字体——这几乎是 99% AI 训练的惯性选择
- 滥用纯黑色背景或文本——缺乏设计质感
- 无效的卡片嵌套与死板的视点尺寸——让布局失去呼吸感
- 过时的缓动曲线——让交互动效呆板生硬

**4. 上下文感知记忆：教会 AI 你是谁**

Impeccable 包含上下文收集协议。初次运行`/impeccable init`时，AI 会启动一轮深度访谈，了解你的项目目标受众、业务场景和品牌个性偏好。这些信息会自动保存在项目根目录的`.impeccable.md`文件中，并在后续所有对话中自动加载，确保 AI 的每一次设计决策都围绕你的独特需求展开，而非套用通用模板。

**5. 产研协同零门槛：内置命令行和浏览器插件**

除了 23 条专业命令外，Impeccable 还提供基于 CLI 和浏览器插件的确定性规则工具集。任何人——包括非技术背景的产品经理和设计师——都可以在不配备 API key 的条件下，运行相同的基础规则扫描，参与视觉质量评审，将设计前移、落地更自然。

**6. 全平台兼容 + 性能卓越**

Impeccable 原生支持 Claude Code、Cursor、Codex CLI、Gemini CLI、VS Code Copilot 和 Antigravity。基于 Skill 架构，安装后代码体积轻量化，启动响应毫秒级，AI 日常交互全程零感知延迟。

> _现实摩擦提醒：早期版本的部分斜杠命令支持需要保持在 Cursor Nightly 预览通道和 Gemini CLI 预览版环境中使用，建议用户注意对应工具的版本稳定性。_

## 03 ｜横向对比

Impeccable 和 Anthropic 官方`frontend-design`技能包的底层逻辑，从一开始就走到两条不同的路上：

| 维度             | **Impeccable**                            | **Anthropic frontend-design** |
| :--------------- | :---------------------------------------- | :---------------------------- |
| **设计深度**     | 7 份完整设计域参考文件 + 语气与品牌自适应 | 1 份基础前端设计准则          |
| **用户干预**     | 23 条专业命令驱动，开发者掌控全局         | 被动触发，默认风格安全但平庸  |
| **规则强化**     | 27 条确定性反模式规则 + 12 条 LLM 审查    | 无，依赖模型固有偏见          |
| **上下文学习**   | 支持项目级品牌语境存储与持续加载          | 无持久记忆，每轮对话重复引导  |
| **核心解决问题** | “开发者不懂设计话术，AI 没有设计概念”     | “给 AI 一点最基础的视觉引导”  |

某种意义上来说，它并不跟同赛道的代码生成工具竞争，而是在重新定义 AI 在这条设计链条里扮演的角色。

当 AI 仅凭基础指引仍会“翻车”时，Impeccable 选择用软硬兼施的策略彻底改造 AI 的思维方式：用 6 万字设计域参考文件重塑专业知识框架，用 23 条控制命令重构交互模式，用确定性反模式规则堵住踩坑死角。

## 04 ｜典型场景和避坑

**适合谁用？**

- **懂技术但不懂设计，却希望团队直接产出高质量 UI 的开发者/全栈工程师**：Impeccable 充当 AI 背后的设计导师，带领模型交出专业级视觉成果。
- **需要给产品经理、市场团队演示高保真原型的 AI 早期探索者**：告别功能粗糙的线框原型，用拟真视觉瞬间捕获观众信任。
- **期望统一团队前端视觉标准的企业和开源项目**：通过 Impeccable 加载共享的设计域文件，让团队每个人的 AI 都遵循同一套设计语言。

**不推荐给谁？**

- **仅需纯后台 API 接口或数据处理逻辑，不涉及 UI 组件的开发团队**：Impeccable 重点落在前端界面层面，后台逻辑用不上它。
- **希望即装即用、不关心设计意图的纯命令式业务团队**：如果不在乎“AI slop”，安装后也无主动使用斜杠指令调度，单纯默认状态下的输出效能提升有限。
- **仍在使用不兼容 Impeccable 的陈旧 IDE 或 AI 插件环境**：先检查官方兼容性列表再做部署决定。

## 05 ｜快速上手指南

Impeccable 的安装方式高度灵活，覆盖从一键全局安装到手动导入的全场景。

**方式一：通用技能安装（最推荐）**

```bash
npx skills add pbakaus/impeccable
```

此命令会自动检测当前环境的 AI 工具类型，完成自动化安装与技能配置。

**方式二：ZIP 包解压（适合离线或手动部署）**

1. 访问`https://impeccable.style`下载通用 ZIP 包
2. 解压 ZIP 包到项目根目录
3. 按需将`.claude/`、`.cursor/`、`.gemini/`等目录中的技能文件拷贝到对应 AI 工具的技能目录

**方式三：Git Clone + 手动添加（适用于 Claude Code）**

```bash
git clone https://github.com/pbakaus/impeccable.git
cd impeccable
claude skill add ./impeccable
```

安装后重启 Claude Code 即可使用所有 Impeccable 内置命令。

**验证安装成功**：在 Claude Code 中输入`/impeccable init`，若进入上下文收集引导流程则安装成功。初次执行`/impeccable init`，系统会自动在项目根目录生成`.impeccable.md`文件并保存品牌偏好，后续交互可随时通过该文件调取持久化记忆。

## 06 ｜总结

昨天你跟 AI 说“做一张定价页”，它给你 Inter 字体+蓝色渐变+卡片嵌套卡片的模板脸。现在你对它说`/impeccable craft`，它给你提一套从字体节奏、色系搭配、空间层次到微动效落地的专业方案。

这不是模型升级了，而是**你给它装上设计大脑了**。

Impeccable 的本质不是写更多代码，而是给 AI 传递一套系统性设计语言——7 份设计域参考文件、23 条专业命令、27 条反模式规则、上下文持久记忆，这些组合在一起，为前端 AI 编写流程注入了一种全新的“设计元直觉”。

对于依然在“设计师资源短缺”和“技术快速迭代”之间挣扎的团队，Impeccable 提供了一种低成本、可规模化的视觉提升路径：无需专业设计背景，懂代码的工程师也能通过精准的控制指令，交付富有质感、气质鲜明的界面产物。

从技术辅助到设计引领的最后一公里，Paul Bakaus 用他沉淀二十年的产品基因，给 AI 编程注入了一剂设计灵魂。
