**大家好，我是雷达君。**

每天在不同的终端之间切换——Hermes Agent 规划任务、Claude Code 写代码、Codex 做原型、Gemini CLI 查资料……你有没有想过：如果有一个统一的 24/7 工作台，能让所有这些 AI 助手一起在线待命，该有多方便？

今天的主角 iOfficeAI/AionUi，正好就是做这件事的。

## 01 ｜核心逻辑

AionUi 的思路非常直白：**为所有 AI 编程助手提供一个永不离线的 Web 操作界面**。

它本质上是一个本地运行的开源 Web 应用，支持：

- 同时挂载 Hermes Agent、Claude Code、Codex、OpenCode、Gemini CLI、Cursor 等 20+ 主流 AI CLI 工具
- 每个助手独立会话、独立上下文
- 24/7 后台运行，不会因为终端关闭而中断
- 完全本地、完全免费、完全开源

想象一下：左边是 Claude Code 在写后端逻辑，右边是 Codex 在画前端原型，下面是 Hermes 在编排整个任务的 pipeline——一切都在一个浏览器标签页里实时可见。

## 02 ｜硬核功能盘点

> 🔗 GitHub 地址：https://github.com/iOfficeAI/AionUi

**多 Agent 并行工作台**
同时跑多个 AI 助手，每个独立会话管理，不会互相干扰。对于复杂的全栈项目，可以并行推进前后端。

**24/7 常驻运行**
基于本地服务模式，不会因为关闭终端而杀掉进程。后台一直跑，第二天打开浏览器继续干。

**20+ CLI 原生支持**
内置对接 OpenClaw、Hermes Agent、Claude Code、Codex、OpenCode、Gemini CLI 等主流工具。即装即用。

**自定义助手**
支持添加自定义 CLI 工具，扩展性极强。如果你有自己的内部工具或 CLI，也可以挂上去。

**完全本地与隐私安全**
数据不出本地，不上云，不开后门。代码全开源，可审计。

## 03 ｜典型场景和避坑

**场景一：全栈项目的多 Agent 协作**
一个复杂的全栈项目，你希望 Claude Code 写后端 Python 逻辑，Codex 写前端 React 组件，Hermes Agent 做任务编排和代码审查。没有 AionUi，你要开三个终端窗口来回切；有 AionUi，一个浏览器完成所有。

**场景二：长期运行的 AI 任务**
比如一个数据迁移脚本要跑几小时，或者一个爬虫要持续监控网页变化。普通终端一关就没了，AionUi 的后台模式让 AI 助手一直在线。

**避坑提醒**：AionUi 是 UI 层，不替代各个 CLI 本身的配置和 API Key 管理。你需要先配置好每个 AI 工具（如 Claude Code 的 API Key、Hermes 的 config），再挂载到 AionUi 上。

## 04 ｜上手教程

**第一步：克隆仓库**

```bash
git clone https://github.com/iOfficeAI/AionUi.git
cd AionUi
```

**第二步：安装依赖 + 启动**

```bash
# 按 README 指引安装
npm install  # 或 yarn install
npm run dev
```

**第三步：挂载你的 AI 助手**
启动后在 Web 界面中添加你的 AI CLI 工具——支持通过 STDIO 或 WebSocket 通信的都可。

**第四步：开始多 Agent 协作**
创建多个会话，分配给不同的 AI 助手，并行推进你的项目。

整个上手过程符合开源项目的标准操作：clone → install → configure → run。

## 05 ｜总结

AionUi 解决了一个非常真实的需求：**当你的 AI 工具箱越来越丰富时，你需要一个统一的操作台**。

它不是又造了一个新 AI 工具，而是把已有的工具串起来，赋予它们 24/7 协作的能力。这种「基础设施思维」比一个孤立的新工具更有长期价值。

对于使用多个 AI 编程助手的开发者和团队来说，AionUi 提供了一种轻量、开源、优雅的方案。
