大家好，我是雷达君。

你有没有遇到过这种尴尬：手上握着一堆 AI 订阅会员——Claude Code、Cursor、GitHub Copilot，甚至还给各个国产大模型充值了好几百。

结果上周五晚上赶项目，催着 AI 改一个棘手的功能，跑了不到一个半小时，Claude Code 的订阅配额先用空了。你停下鼠标点开配置面板，Key 复制黏贴了三轮才切成另一个提供商。最抓狂的是，刚切过去跑了 20 分钟，**配额又没了**。

问题不是钱没花够，而是**钱花得太散了**。

GitHub Copilot 每月留着大把没用完的 token，GitHub Models 天天有免费额度到期作废，国产低价提供商买了套餐，总在角落里落灰。每次想充分利用这些资源，就得手动切换配置——改 host、换 API Key、调环境变量……切到第三个方案的时候，你已经完全不想跟 AI 继续沟通了。

今天雷达君要安利的这个开源项目，就是为了解决这个问题而生的——一个可以本地自部署的**AI Agent 网关**，在你本机上提供一个统一的流量出入口。前面可以接 Claude Code、Codex、OpenCode、OpenClaw 等所有 AI 编程工具，后面连接你手里的一堆订阅和 API 提供商，靠智能策略把它们统一榨干用尽。

它就是——**9Router**。

## 01 ｜核心逻辑

9Router 的核心思路并不复杂：**把你手中所有的 AI 资源，变成一套统一的 AI 生产力。**

它在你本地跑一个代理服务，对外暴露一个统一的 OpenAI 兼容 API 端点。所有 AI 编程工具都指向这个地址，由 9Router 帮你决定“这次请求应该打到哪家提供商”。

支撑它跑起来的，是几套环环相扣的核心能力：

**三层自动回退**：依照 **订阅配额 → 廉价 API → 免费额度** 的优先级自动切换。当前渠道用完或报错，立刻无缝切到下一层，编码过程不被打断。

**多账号轮询**：同一个提供商支持绑定多个账号，自动轮流使用。用三个不同订阅的 GitHub Copilot OAuth 组成一个“Combo”池子，请求在它们之间轮询，就能做到等效无限吞吐量，永远不会在单个账号上碰速率极限。

**RTK Token 压缩**：在工具返回结果（git diff、grep 结果、ls 输出的海量目录树）交给大模型之前，提前做结构性压缩，节省 20-40%的输入 Token。

**Caveman 原始人模式**：系统级 Prompt 重写——把啰嗦的指令精简到“要简洁、不要 Markdown”的硬核风格，输出 Token 最多可减少 65%。

**实时配额追踪**：对每个提供商、每个模型单独跟踪用量和重置倒计时。花钱买的订阅额度在月底归零之前被彻底榨干。

**OAuth 动态刷新**：那些走 OAuth 登录的免费提供商（如 Kiro 无限 Claude 额度、GitHub Copilot 订阅），9Router 能自动帮你刷新过期 Token，不用人工反复拷贝。

顺带提一句：极少数专注于多账号 Token 轮换的修改版确实存在，但那属于极客个人的改造探索方向，并非 9Router 主分支的官方行为，不过这也反过来证明了这套架构的开放性和可扩展性。

## 02 ｜硬核功能盘点

**1. 40+提供商、100+模型的一站式接入**

9Router 背后对接的提供商名单非常夸张：GitHub Copilot 的 OAuth、Gemini CLI 免费会话、本地 Ollama、Cursor、Codex、Kiro（提供免费无限 Claude 额度的 AWS 服务）、谷歌 Vertex、OpenCode Free、国产低价 API（GLM、MiniMax、Kimi）……算下来已经有 40 多家。几乎所有市面上出现过的 AI 模型，都能在这个池子里找到接入点。

**2. 通用性极高：任何 AI 工具都能接**

不管是 Claude Code 还是 Codex，不管是 Cursor、Cline、Continue 还是 GitHub Copilot——只要工具支持 OpenAI 兼容 API，或者支持修改`ANTHROPIC_BASE_URL`和`OPENAI_BASE_URL`这两个环境变量，通通可以把请求指向 9Router。

**3. 智能 Combo 组合：多订阅无限吞吐**

这可能是 9Router 设计中最骚的一个点——你可以把多个同类型提供商的账号捏成一个“Combo”组合，选轮询（request 轮流使用不同账号）或者粘性轮询（一个会话始终用一个账号）。三个不同订阅组合在一起，单个账号的速率限制就不再是天花板，三个轮流跑的时候就好像没有限制一样。

**4. 友好可视化仪表盘 + 安全配置同步**

9Router 自带一个完整的 Web 管理后台（默认运行在`http://localhost:20128/dashboard`），提供商管理、端点创建、用量分析全部可视化的点一下就行。配置还支持跨设备云端同步，也可以把 JWT_SECRET 和 INITIAL_PASSWORD 交给团队关键成员共同维护。

**5. 请求日志 + 用量分析**

你可以在仪表盘里清晰追踪每轮对话的 Token 消耗、成本统计、上下行流量趋势图。调试模式下还可以看到完整的请求/响应记录。

**6. MITM 反重力代理：撬动 VS Code Copilot 的唯一方案**

针对 VS Code 的 Copilot 扩展，9Router 提供了一个“反重力”（Antigravity）MITM 代理模式——在本地 443 端口做 HTTPS 中间人拦截，把原来只能用 Copilot 后端的流量强行切到 9Router 支持的所有提供商池子里，同时也支持对齐 Claude Code 等工具的对话。**但这个模式需要你手动安装自签名证书，需要在系统层级信任 CA，还会用到 sudo 命令，属于“知道自己在干什么”的黑客式操作**，普通用户谨慎使用。

## 03 ｜与技术生态对比

9Router 和市面上其他 AI 网关有一个根本性的设计取舍不同：**它不是给整个团队做企业级接入用的，而是为一个重度使用 AI 编程工具的个体开发者，把手上一堆订阅、免费额度榨干而设计的**。

- **LiteLLM**：Python 写，企业级多 Provider 网关，强项是审计、成本追踪、限流，适合 K8s 上给上百人用。
- **One API / New API**：中文圈最多人用的 LLM 统一网关，主打“给整个团队用”的计费与密钥管理。
- **OpenRouter**：SaaS 路由器，自己不托管模型，纯做计费聚合。
- **ClawRouter**：为 AI Agent 设计的智能路由器，分析 15 个维度自动选模型，支持 USDC 加密货币微支付，底层逻辑与 9Router 完全不同，前者走区块链生态路线，9Router 走本地免费资源聚合路线。
- **9Router**：偏“个人开发者本机”，重点在订阅配额追踪、OAuth 自动刷新、针对 CLI 工具的 Token 压缩。

一些小众分支（如聚焦于账户自动轮换的 n9router 分支）的改动也值得留意，但它偏离了 9Router 主分支的核心设计，只是说明这套架构有社区改造的灵活性。

## 04 ｜典型场景和避坑

**极适合的场景：**

- **手上有多个不同 AI 提供商套餐、饱受配额度卡脖子之苦的重度 AI 用户**：9Router 把所有 AI 工具统一接进来，智能路由策略帮你统筹额度分配。
- **想零花费跑通高端模型（Claude/GPT/Gemini）的个人开发者/学生**：Kiro 免费无限 Claude + GitHub Models 免费额度 + OpenCode Free 免费资源，在 9Router 里自动轮询。
- **希望在 Claude Code、Cursor、Cline 等多工具间共享一套 AI 资源池**：一套 API Key 在所有工具里通用，不用每切一次换配置。

**不推荐/需要谨慎考虑的场景：**

- **只用一个 AI 提供商且用量不大**：加一层代理会引入几十毫秒的延迟，不值当。
- **做生产级、安全关键系统的付费企业团队**：生产环境部署前必须做分阶段压力测试、加强凭证管理和日志审计。而 9Router 的账号管理和健康监控等环节的功能成熟度也不足以支撑企业级审计。
- **对 compression 要求极高的安全场景**：RTK 压缩依赖第一个 1KB 快速检测模式，在某些极端 token 爆炸场景下可能测不准。

## 05 ｜快速上手指南（3 分钟）

**方式一：npm 一键安装（最推荐）**

```bash
npm install -g 9router
9router
```

浏览器自动打开仪表盘 `http://localhost:20128/dashboard`，完成初次启动。

**方式二：npx 无需安装直接跑**

```bash
npx 9router
```

适合临时测试或不想全局污染环境的场景。

**方式三：Docker 跑（适合长期挂载）**

```bash
docker run -d --name 9router -p 20128:20128 -v ~/.9router:/root/.9router decolua/9router
```

。

**启动后接入你的 Claude Code**：在仪表盘生成一个 API Key，然后设置代理指向 9Router：

```bash
export ANTHROPIC_BASE_URL=http://127.0.0.1:20128
export ANTHROPIC_API_KEY=9r_yourkeyhere
```

回到终端执行 Claude Code 命令，所有请求就会自动走 9Router 的路由路线。

**给国内同胞的一点实践提示**：在添加提供商的时候，如果遇到 Minimax 国内版连不上的情况，可以切换到 Minimax(China)选项；Mimo 目前开箱即用暂时有问题，但可以使用页面底部的“添加 OpenAI 兼容”选项来自定义配置。这套调整方案是中文社区的实践经验总结。

## 06 ｜总结

9Router 解决的核心问题是什么？

不是你买不起 AI 模型，而是你花钱买了太多零散的 AI 资源，**它们被锁在不同的生态系统里各自为战**。

Claude Code 有它的配额墙，Cursor 有它的积分体系，GitHub Copilot 自成一套循环。每个 AI 工具都想让你在它的闭环里付费。9Router 就是那个主动刺破这些闭环的工具——它用一个统一网关把你所有的 AI 资源拉平成一整张池子，智能回退和三模式路由编队随时帮你把每一分花出去的订阅费都用到位。

最宝贵的东西不是钱，而是写着写着配额度报错，思路断掉、切来源重配的那十几分钟，9Router 把这些损耗全部省掉了。

GitHub 6.5K Star 可以代表社区的认可，但真正的价值，还是那句老话：别让写在配置文件上的钱，浪费在额度重置的路上。
