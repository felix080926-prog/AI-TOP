## 01. 核心逻辑：Your Login is the API

以前我们要让 AI 抓取数据或者操作网页，最头疼的就是“登录态”。

  * 提取 Cookie 容易失效；
  * 模拟登录会被验证码卡死；
  * 官方 API 又贵限制又多。



**Onyx 极其聪明地避开了这些坑：它直接调用你本地的浏览器 Session。**

简单来说，只要你在 Chrome 里登录了账号，Onyx 就能直接“接管”。它不需要你折腾 API Key，你的浏览器就是它最强的 API。

* * *

## 02. 为什么 Onyx 敢说自己是 Web Agent 的新标杆？

如果你仔细看它的源码（GitHub: `onyx-dot-app/onyx`），你会发现它叠了三层极硬核的 Buff：

### ** Buff 1：视觉驱动（VLM-Native）**

传统的自动化脚本（比如早期的 Selenium）如果遇到网页改版，定位符一变就直接报错。但 Onyx 是看“截图”的。它利用 GPT-4o 或 Claude 3.5 的多模态能力，像人眼一样识别“搜索框在哪”、“登录按钮在哪”。就算网页 UI 天天变，它也能照样操作。

### ** Buff 2：MCP 生态的降维打击**

它深度集成了 **Model Context Protocol（MCP）** 。这意味着它不是孤军奋战，而是自带了一个拥有 100+ 热门平台（微博、知乎、小红书、GitHub、京东...）适配器的“技能包”。AI 不再需要费劲去解析 HTML 网页源码（那太费 Token 了），直接调用结构化指令，效率高得惊人。

### ** Buff 3：本地优先（Local-first）**

这可能是我最看重的一点。所有的自动化操作和数据流转都在你的本地 `localhost` 完成。你的账号信息、浏览记录、隐私数据绝不上传云端。这种安全感，是那些在线 Agent 平台给不了的。

## 04. 怎么部署？（开发者 1 分钟上手）

你不需要深的 Linux 知识，只要你有 Python 环境，一行指令领养你的“数字员工”：
    
    
    pip install onyx playwright  
    python -m onyx setup  
    

配置好你的 LLM 密钥（推荐用 Claude 3.5 Sonnet，视觉理解力最强），然后你就能看到一个无头浏览器窗口自动打开，AI 像个老师傅一样开始在上面“冲浪”。
