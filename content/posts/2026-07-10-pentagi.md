**大家好，我是雷达君。**

你有没有经历过这种场景：产品上线前，安全团队跑了一遍漏扫工具，报告里躺着一堆“疑似漏洞”。你花了一整天排查，最后发现一半是误报，另一半虽然是真的，但安全团队已经累得没力气验证了。

更麻烦的是，真正的 0day 漏洞和逻辑漏洞，扫描器根本扫不出来。你得请外部渗透测试公司，排队两周、报价五万起步，等报告出来黄花菜都凉了。

安全团队永远缺人，渗透测试永远排期，漏洞永远在线上等着被黑客发现。

**网络安全的最大瓶颈，从来不是工具不够多，而是能看懂工具输出、能串联攻击链路的人太少了。**

今天雷达君要安利的这个开源项目，就是来打破这个瓶颈的。它是一个**全自主 AI 渗透测试系统**——沙箱隔离、20+专业工具内置、知识图谱记忆、多专家 Agent 协作，能像人类渗透测试工程师一样自主完成侦察 → 漏洞发现 → 利用验证 → 报告生成的全链路。

它就是——**PentAGI**。

## 01 ｜核心逻辑：给安全团队配一支“永不下班的 AI 黑客军团”

PentAGI 的核心理念可以用一句话概括：**让 AI 像人类渗透测试专家一样思考，像自动化工具一样执行。**

传统的渗透测试怎么做？安全工程师打开 Nmap 扫端口，看到开放服务后用 Metasploit 尝试利用，手工验证漏洞，再写报告。每一步都需要人盯着、判断、决策。一个中等规模的应用，完整的渗透测试至少需要 2-3 周。

PentAGI 换了一套打法。它派出一个由**多个专业 AI Agent 组成的“黑客军团”**：

- **Research Agent（研究专家）** ：负责情报收集和攻击面分析
- **Development Agent（开发专家）** ：负责编写和调试漏洞利用代码
- **Infrastructure Agent（基础设施专家）** ：负责管理 Docker 容器和工具链

这些 Agent 各司其职、协同作战——Research Agent 发现一个可疑端口，自动通知 Development Agent 去尝试利用，Infrastructure Agent 在沙箱里拉起对应的工具容器。整个过程**全自动、可监控、可追溯**。

PentAGI 的设计团队把它定位为“**自主渗透测试平台**”，而不是“BAS（入侵与攻击模拟）产品”。它不预置攻击剧本，而是让 AI Agent 自己决定攻击路径——更像一个真实的渗透测试工程师，而不是一个执行预设脚本的自动化工具。

**一句话：别人给你一堆扫描报告，PentAGI 给你一支能自主挖洞、验证、出报告的 AI 渗透测试团队。**

## 02 ｜硬核功能盘点

> 🔗 **GitHub 地址**：https://github.com/vxcontrol/pentagi

### 1. 20+专业安全工具内置：Nmap、Metasploit、SQLmap……一把梭

PentAGI 内置了一套完整的渗透测试工具链，涵盖侦察、漏洞利用、Web 安全等全链路：

- **网络扫描**：Nmap、Masscan
- **漏洞利用**：Metasploit、Searchsploit
- **Web 安全**：SQLmap、Burp Suite、Nikto
- **密码攻击**：John the Ripper、Hydra
- **流量分析**：Wireshark、tcpdump
- **其他**：Gobuster、FFuf、Recon-ng 等

这些工具全部运行在**Docker 沙箱**中，与主机完全隔离。AI Agent 在沙箱里执行任何操作——哪怕是跑`rm -rf /`——也只影响容器内部，你的主机和数据毫发无伤。

### 2. 四层记忆系统：AI 越用越“懂”你的业务

PentAGI 设计了完整的记忆架构，让 AI Agent 不是“用完就忘”：

- **长期记忆**：持久化存储知识和经验
- **工作记忆**：当前操作的上下文和目标
- **情景记忆**：历史操作和成功模式
- **知识库**：结构化的领域专业知识和工具能力

这些记忆存储在**PostgreSQL + pgvector**中。AI Agent 下次遇到类似目标，会直接调用之前的成功经验，而不是从零开始。

### 3. 知识图谱集成：让 AI 理解攻击之间的“因果关系”

PentAGI 集成了**Graphiti + Neo4j**知识图谱。传统的扫描器给你一份扁平的报告——IP、端口、漏洞、CVE 编号。PentAGI 的 AI Agent 能看到攻击之间的**语义关系和因果链条**：

- 端口 A 开放 → 服务 B 运行 → 存在漏洞 C → 可以被利用 D
- 漏洞 E 被利用后 → 可以访问内网 F → 进而攻击目标 G

**这就像给 AI 装了一个“攻击链路地图”**，而不是一堆零散的“路标”。

### 4. 外部搜索引擎集成：情报收集能力拉满

PentAGI 集成了**7 种外部搜索系统**：

- **Tavily、Traversaal、Perplexity**：AI 驱动的搜索引擎
- **DuckDuckGo、Google Custom Search**：传统搜索引擎
- **Sploitus**：漏洞利用数据库搜索引擎
- **Searxng**：元搜索引擎

AI Agent 在侦察阶段会自动调用这些搜索引擎，收集目标的最新情报——CVE 公告、漏洞利用代码、技术博客、论坛讨论——然后把这些信息整合进攻击决策中。

### 5. 10+ LLM 提供商 + 自定义端点：模型自由，想用哪个用哪个

PentAGI 支持**10+ LLM 提供商**：

- OpenAI、Anthropic、Google Gemini、AWS Bedrock
- DeepSeek、GLM、Kimi、Qwen
- Ollama（本地部署）
- 聚合器：OpenRouter、DeepInfra
- **自定义 OpenAI 兼容端点**

你可以用云端最强模型跑复杂推理，也可以用本地 Ollama 跑低成本任务。对于生产级本地部署，PentAGI 还提供了**vLLM + Qwen3.5-27B-FP8 的完整部署指南**。

### 6. 完整的 Web UI + REST/GraphQL API

PentAGI 提供了一个**现代化 Web 管理界面**，你可以在浏览器里：

- 创建和管理渗透测试任务
- 实时查看 Agent 执行状态
- 配置 LLM 提供商和搜索系统
- 查看和导出漏洞报告

同时提供**完整的 REST 和 GraphQL API**，支持 Bearer Token 认证。你可以把 PentAGI 集成到 CI/CD 流水线、安全运营平台或内部工具链中。

### 7. Langfuse 集成 + Grafana/Prometheus 监控

PentAGI 原生集成**Langfuse**，提供全链路的 LLM 调用追踪和可观测性。同时支持**Grafana + Prometheus**实时监控，你可以实时看到系统状态、Agent 执行进度和资源使用情况。

**你的 AI 黑客军团不仅会干活，还会“汇报工作”。**

## 04 ｜典型场景和避坑

**适合谁用？**

- **需要频繁进行安全测试但人力不足的安全团队**：PentAGI 的 AI Agent 可以 7×24 小时自主执行渗透测试，把安全专家从重复劳动中解放出来
- **DevSecOps 团队**：把 PentAGI 集成到 CI/CD 流水线，每次代码合并自动触发安全测试
- **需要定期出渗透测试报告的企业**：PentAGI 能生成详细的漏洞报告和利用指南
- **安全研究者和红队成员**：PentAGI 可以作为自动化侦察和漏洞验证的辅助工具

**不推荐给谁？**

- **只需要简单端口扫描的轻量场景**：直接跑 Nmap 可能更快
- **完全没有 Docker 和 Linux 运维经验的纯安全分析师**：PentAGI 的部署和维护需要一定的技术基础
- **需要在没有互联网接入的完全隔离环境中运行**：PentAGI 需要下载 Docker 镜像和 LLM API 调用

> **现实摩擦提醒**：PentAGI 目前是“自主渗透测试平台”，而不是“BAS（入侵与攻击模拟）产品”。它不预置攻击剧本，而是让 AI Agent 自主决策攻击路径。这意味着它的输出质量高度依赖 LLM 的能力和配置——用弱模型跑复杂目标，效果可能会打折扣。官方推荐参考 vLLM + Qwen3.5-27B-FP8 的生产级部署指南。

## 05 ｜快速上手指南（5 分钟）

PentAGI 的安装非常友好，官方提供了**交互式安装器（Installer）** ，全程引导你完成配置。

**环境要求**：Docker、2 vCPU、4GB RAM、20GB 硬盘

**第 1 步：下载安装器**

```bash
# Linux amd64
mkdir -p pentagi && cd pentagi
wget https://pentagi.com/downloads/linux/amd64/installer-latest.zip
unzip installer-latest.zip
```

Windows、macOS（Intel/M 系列）用户可从官网下载对应版本。

**第 2 步：运行交互式安装器**

```bash
sudo ./installer
```

安装器会依次引导你完成：

1. **系统检查**：验证 Docker、网络连通性和系统要求
2. **环境配置**：创建`.env`文件并设置最优默认值
3. **LLM 提供商配置**：选择 OpenAI、Anthropic、Gemini、Bedrock、Ollama 或自定义
4. **搜索引擎配置**：配置 DuckDuckGo、Google、Tavily 等
5. **安全加固**：生成安全凭证和 SSL 证书
6. **部署启动**：用 docker-compose 启动 PentAGI

**第 3 步：访问 Web UI**

浏览器打开`http://localhost:3000`，登录后即可开始创建渗透测试任务。

**验证安装成功**：在 Web UI 中创建一个测试任务，如果 Agent 能正常启动并执行侦察步骤，说明安装成功。

> **生产环境建议**：对于生产部署或安全敏感环境，官方强烈推荐使用**分布式双节点架构**——主节点运行 Web UI 和调度，工作节点运行隔离的 Docker 容器执行渗透测试。这可以防止不受信任的代码执行和网络访问影响到你的主系统。

## 06 ｜总结

PentAGI 解决的核心问题是什么？

**网络安全的最大瓶颈，从来不是工具不够多，而是能看懂工具输出、能串联攻击链路的人太少了。**

Nmap 会扫端口，Metasploit 会打漏洞，SQLmap 会注数据库——但把这些工具串联成一条完整的攻击链路、并根据现场情况动态调整策略，只有人类渗透测试工程师能做到。而这个工种，永远供不应求。

PentAGI 的答案是用 AI 多 Agent 系统复刻这个“人类渗透测试工程师”的能力。Research Agent 做情报收集，Development Agent 写利用代码，Infrastructure Agent 管工具链——三个 AI 专家协同作战，在 Docker 沙箱里安全地完成侦察 → 漏洞发现 → 利用验证 → 报告生成的全链路。

20+专业工具内置、四层持久记忆、知识图谱、7 种搜索引擎、10+ LLM 提供商、完整的 Web UI 和 API——PentAGI 给安全团队配的是一支“永不下班的 AI 黑客军团”。
