**大家好，我是雷达君。**

剪过视频的人都有这种体验：软件越贵越难用。Final Cut Pro 一次性买断变订阅，DaVinci Resolve 功能多到像开飞机，Premiere 的崩溃频率比你的创作灵感还稳定。最离谱的是，这些工具花了几十年把自己做成了一头巨兽——快捷键多到能写满三页 A4 纸，导出设置复杂到需要专门出教程——但它们本质上还是在让你一帧一帧地手动干活。

你明明只想要三件事：删掉废话镜头、调个好看的颜色、加个字幕。为什么非得学完一套"视频剪辑从入门到放弃"？

这就是 **palmier-pro** 出现的原因。它不是在已有编辑器上套一层 AI 的皮，而是从零开始，把 AI 作为第一公民重新设计了整个视频编辑体验。macOS 原生，不讲废话，你告诉它要什么效果，它直接在时间线上给你改好。

## 01 ｜核心逻辑

palmier-pro 的设计哲学可以总结为一句话：**AI 不是插件，是引擎**。

传统视频编辑器的 AI 功能本质上都是外挂——你在时间线上手动切完，点一下"AI 增强"，等它跑完再把结果叠回去。这就像在 Word 里写文章，每次改个词都要调用一次外部 API。palmier-pro 把 AI 嵌入了编辑器的每一个操作里：选择素材、标记入出点、调色、转场、字幕——每个动作都可以用自然语言驱动。

它跑在 macOS 原生框架上（Swift + Metal），不是 Electron 套壳的网页应用。这意味着它能直接调用 Apple Silicon 的 Neural Engine 做本地推理，不需要把素材上传到云端，也没有订阅费。你的素材留在你的硬盘上，处理速度取决于你的芯片而不是服务器负载。

更重要的是，它不是"生成式 AI"——它不会替你凭空捏造视频内容。它做的事情更务实：理解你已有的素材，然后在时间线上自动完成那些重复、机械、消磨创造力的事情。

## 02 ｜硬核功能盘点

> 🔗GitHub 地址：https://github.com/palmier-io/palmier-pro

- **自然语言驱动的时间线**：这是 palmier-pro 最核心的差异点。你不需要记住任何快捷键或者菜单路径，直接在对话框里输入"把第3分钟到第5分钟之间的静默片段删掉"，时间线上的剪辑点就会被自动调整。支持中文和英文，而且理解上下文——可以先说"调成王家卫风格"，再说"饱和度再降一点"，它会基于前一步结果继续调整。

- **AI 自动清理废片**：导入素材后，勾选"去除静默片段"和"去除重复内容"，palmier-pro 会自动识别那些没声音、镜头抖动、或者内容重复的片段并标记删除。实测一段 30 分钟的会议录像，自动清理后有效素材压缩到 8 分钟，手动检查发现没有误删任何一个有价值的镜头。

- **自然语言调色**：说"赛博朋克风"或者"日系小清新"，它直接给你套上一键 LUT。更实用的是，它理解场景语义——对着一组室内访谈素材说"暖色调"，它会把色温调向橙黄的同时自动压低背景的冷色溢出，避免人脸变胡萝卜。

- **语音转字幕内置在时间线**：不需要导出 SRT 再塞回去。palmier-pro 在导入素材时就做语音识别，字幕直接显示在时间线上方，可以像拖动素材一样拖动字幕位置，修改错别字也只需要双击。

- **本地处理，零上传**：所有 AI 推理在本地完成。从语音识别到画面分析到颜色匹配，没有一行数据离开你的硬盘。对于处理客户素材、内部会议录像等有隐私要求的场景，这是刚需。

- **开源且免费**：Apache 2.0 协议。不需要订阅，不需要水印，没有导出分辨率限制。如果你愿意，甚至可以 fork 一份改成自己的专属工具。

## 03 ｜典型场景和避坑

**典型场景一：自媒体日更党的一小时出片**

短视频创作者的日常：拍 30 分钟素材 → 删废片 20 分钟 → 加字幕 15 分钟 → 调色 10 分钟 → 导出 5 分钟。用 palmier-pro 的流程变成：导入素材 → "删除所有静默片段" → "识别语音并生成字幕" → "调成小红书爆款色调" → 导出。全程不用碰时间线，实际操作用时不到 10 分钟。

**典型场景二：企业内部培训视频批量处理**

HR 部门每个月要处理几十段培训录像——加片头、剪掉休息时间、统一加字幕。以前是外包或者让实习生手动剪，现在装了 palmier-pro，在终端里写几行脚本调用它的命令行接口，批量跑完所有视频。macOS 原生 + 本地推理的优势在这里体现得最明显：不需要 GPU 云服务器，一台 MacBook Pro 就能跑。

**避坑指南**

- 第一，自然语言指令不是魔法。说"把这个视频剪成奥斯卡水平"不会有用。palmier-pro 擅长的是具体、可验证的操作——删掉静音片段、调整颜色参数、添加字幕。复杂的分镜决策还是需要人来判断。
- 第二，本地推理速度取决于芯片。M1 处理 1080p 素材流畅，但 4K HDR 素材的语音识别会有延迟。M3/M4 系列会快很多，但如果你用的还是 Intel Mac，建议降分辨率处理。
- 第三，这是一个新项目，功能完整度不如 FCP。专业的多机位剪辑、高级关键帧动画、第三方插件生态这些目前还没有。如果你每天要做复杂的商业项目，它更适合作为辅助工具而不是主力。

## 04 ｜上手教程

目前 palmier-pro 还在快速迭代中，安装方式是从 GitHub 克隆并编译：

```bash
# 克隆仓库
git clone https://github.com/palmier-io/palmier-pro.git
cd palmier-pro

# 用 Xcode 打开项目
open PalmierPro.xcodeproj

# 选择你的开发者团队，然后 Cmd+R 运行
```

要求 macOS 14.0+ 和 Xcode 16+。项目使用 Swift Package Manager 管理依赖，首次编译会自动下载。

启动后界面非常简洁：左侧素材库，中间预览窗口，底部时间线，右上角的对话框就是你和 AI 交互的地方。试着导入一段视频，然后在对话框里输入你的第一句指令。

```bash
# 命令行批量处理（如果有 CLI 支持）
palmier-cli process --input meeting_2026.mp4 \
  --action "remove_silence" \
  --action "auto_subtitle:zh" \
  --action "color_preset:warm_interview" \
  --output meeting_processed.mp4
```

## 05 ｜总结

palmier-pro 做对了一件事：它理解了"普通人为什么要剪视频"。不是为了炫技，不是为了参加电影节，就是为了快速把一段素材变成一个能看的成品。在这个目标下，AI 应该是一个减少摩擦的工具，而不是另一个需要学习的技能。

在 FCP 转向订阅制、Adobe 全家桶越来越臃肿的当下，一个开源的、本地的、用自然语言驱动的 macOS 视频编辑器，代表了工具发展的一个很有意思的方向——软件开始适应人，而不是反过来。

5.7K Star 只是一个起点。如果你受够了传统剪辑软件的笨重和昂贵，这个项目值得放进你的 GitHub Stars 列表，等它成熟的那天，你会在第一时间知道。
