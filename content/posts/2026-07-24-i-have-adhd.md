**大家好，我是雷达君。**

你刚问完 AI 一个问题，它开口就是一句“Great question!”。紧接着，它会贴心地告诉你，你的问题“有几个环节”需要考虑，然后逐条分析，最后还不忘补一句“Hope this helps!”。整段话行云流水，词藻华丽，你手指滚了三次轮，才终于在倒数第二段找到了那个自己需要的 npm 命令。

那一刻，AI 不是来帮你解决问题的，是来帮你磨炼耐心的。然而，AI 并不知道，它的这份“礼貌”，在很多人眼里，就是实打实的干扰。开发者@ayghri 不仅意识到了这个问题，还把它做成了一个开源项目。

**这个项目叫——`i-have-adhd`**。

## 01 ｜核心逻辑：AI 看不懂“多动症”，但能学会“不废话”

> 🔗 GitHub：https://github.com/ayghri/i-have-adhd

首先说清楚：你不需要真的有 ADHD（注意力缺陷多动障碍）才能用这个项目。它本质上是一套严格到近乎苛刻的**输出格式约束规则**。你把它装进 AI 编程助手（比如 Claude Code 或 Codex）里，它会立刻让 AI 的“嘴”变短，让 AI 的“手”变快。

普通的 AI 回答像是在写一篇散文——有开场、有铺垫、有高潮、有升华。而装了 `i-have-adhd` 的 AI，回答起来像在写**操作手册**——先说结论，第二步编号，第三步结束，没有一句废话。

这背后，是 10 条非常具体的规则在起作用：

1.  **先说下一步行动**：第一句话必须是读者能做的具体操作，不是介绍背景，不是分析利弊。
2.  **多步骤任务使用编号**：每个步骤都是一次性的具体操作，不写“然后”。
3.  **以一个具体的下一步结束**：如果还有遗留问题，只说一件能在两分钟内完成的事。
4.  **抑制离题**：做完一件事，再问下一件事。不跑题，不顺便。
5.  **每轮都重述状态**：因为工作记忆很有限，不能指望对方“记住”上下文。
6.  **给出明确的时间估计**：“一会儿”这种模糊词全部消失，只说“几分钟”。
7.  **让成果清晰可见**：让进展可视化，不把“做完”这个信号埋起来。
8.  **客观陈述错误**：陈述错误时只说事实，不加情绪。
9.  **每个列表最多 5 项**：不列长清单，避免信息过载。
10. **不写开场白、回顾或结束语**：不寒暄、不总结、不客套。

## 03 ｜怎么装上这个“注意力过滤器”？

安装非常简单，不需要克隆仓库，AI 编程助手会自动获取并保持更新。

如果你在用 **Claude Code**，在终端里依次执行：

```bash
claude plugin marketplace add ayghri/i-have-adhd
claude plugin install i-have-adhd@i-have-adhd
```

然后在对话中输入 `/i-have-adhd` 即可激活。

如果你在用 **Codex**，执行：

```bash
codex plugin marketplace add ayghri/i-have-adhd --ref main
codex plugin add i-have-adhd@i-have-adhd
```

然后输入 `$i-have-adhd` 显式激活。

**想让它在每次会话中都自动启用？**

```bash
touch ~/.claude/.i-have-adhd-always
```

一次配置，终身生效。

想恢复正常模式，直接说“stop adhd mode”或“normal mode”就行。

## 04 ｜“治疗”原理：来自心理学，用于 AI

这套规则并非凭空捏造，而是参考了 J. Russell Ramsay 和 Anthony L. Rostain 合著的**《The Adult ADHD Tool Kit》**（成人 ADHD 工具包）。

作者把针对 ADHD 患者的沟通策略——**减少认知负荷、提供清晰步骤、降低启动门槛、给予即时反馈**——完整移植到了 AI 助手的输出逻辑上。

它的本质不是给 AI“看病”，而是给 AI 的**输出格式做了一次彻底的“认知友好型”改造**。

## 05 ｜总结

`i-have-adhd` 解决了一个非常具体的问题：**AI 的回答太“客气”了，客气到耽误事。**

“Great question!”和“Hope this helps!”在传统对话里是礼貌，在 AI 辅助编程的场景里，就是占用你注意力的噪音。`i-have-adhd` 把 AI 从“一个懂技术的客服”变成了“一个坐在你旁边的技术搭档”——他话不多，只告诉你现在该干什么。

安装包和完整代码都在这里：https://github.com/ayghri/i-have-adhd
