**大家好，我是雷达君。**

你有没有过这种体验：让 Claude 帮你解释一个 React 组件重渲染的原因，它洋洋洒洒写了 69 个 token 的完整分析。你等它输出完，核心信息其实就一句话——**“内联对象每次都是新引用，导致重渲染。”**

剩下的 50 多个 token，全是“I'd be happy to help”“The reason is likely because”“I'd recommend using”这类礼貌性填充。

不是 AI 不聪明，是它太礼貌了。每个回答都要先客套两句，再详细展开，最后再总结一遍。你问一个简单问题，它给你一篇小作文。

更扎心的是，在 Claude Code 这种按 Token 计费的工具里，你为这些客气话付了真金白银。

如果能让 AI 像**原始人**一样说话——砍掉所有客套、废话、重复，只保留最核心的技术信息呢？

今天雷达君要安利的这个开源项目，就是在做这件事。它由开发者 Julius Brussee 打造，是一个让 AI“说人话”的插件——准确地说，是说**“原始人话”**。

它就是——**Caveman**。

## 01 ｜核心逻辑：脑子还是那个脑子，只是嘴变小了

Caveman 的名字很直白——“原始人”。它做的事情也只有一件：**让 AI 像原始人一样说话**。

它不是让 AI 变笨。Caveman 的官方定位非常清晰：**“Brain still big. Mouth small.”**——脑子还是那个脑子，只是嘴变小了。

具体来说，Caveman 是一个**Claude Code 技能/插件**，同时也支持 Codex、Gemini、Cursor、Windsurf、Cline、Copilot 等**30 多种 AI 编程工具**。安装后，AI 在回答时会自动：

- **删掉所有客套话**——“I'd be happy to help”“Sure! Let me take a look”全部消失
- **砍掉重复解释**——不说两遍同样的事
- **保留技术干货**——代码、命令、错误信息、技术判断一字不差
- **用短句代替长文**——片段式输出，直击要害

一个真实的对比：同样解释 React 重渲染问题，正常 Claude 用了 69 个 token，Caveman 模式只用了 19 个 token。信息量完全一样，Token 消耗少了**72%**。

**Caveman no make brain smaller. Caveman make mouth smaller.**

> _值得一提的是，Caveman 的作者 Julius Brussee 还打造了完整的 Caveman 生态——除了输出压缩的 caveman，还有完整终端编码代理 caveman-code、跨 Agent 记忆 cavemem、规范驱动构建循环 cavekit，以及微调了 Caveman 对话风格的 cavegemma 模型。五个工具共用同一哲学：Agent 用更少资源做更多事。本文聚焦的 caveman 是其中 Star 数最高、传播最广的基石项目。_

## 02 ｜硬核功能盘点

> 🔗 **GitHub 地址**：https://github.com/JuliusBrussee/caveman

### 1. 四种压缩模式：从“精简”到“文言文”，按需切换

Caveman 提供了四个级别的压缩强度，你可以随时切换：

- **`lite`**：轻度精简，只删明显的填充词
- **`full`**（默认）：标准原始人模式，砍掉大部分废话
- **`ultra`**：电报式输出，极致压缩
- **`wenyan`**：文言文模式——比 ultra 还要短

一条命令就能切换：`/caveman ultra`，之后的回答全部按新级别压缩。

### 2. 真实数据：平均砍掉 65%输出 Token

Caveman 在 README 里放了一组详细的基准测试数据：

| 任务                      | 正常 Token | Caveman | 节省    |
| :------------------------ | :--------- | :------ | :------ |
| 解释 React 重渲染 Bug     | 1180       | 159     | **87%** |
| 修复认证中间件 Token 过期 | 704        | 121     | **83%** |
| 配置 PostgreSQL 连接池    | 2347       | 380     | **84%** |
| 解释 git rebase vs merge  | 702        | 292     | **58%** |
| 重构回调为 async/await    | 387        | 301     | **22%** |
| 微服务 vs 单体架构        | 446        | 310     | **30%** |
| PR 安全审查               | 678        | 398     | **41%** |
| Docker 多阶段构建         | 1042       | 290     | **72%** |
| 调试 PostgreSQL 竞态条件  | 1200       | 232     | **81%** |
| 实现 React 错误边界       | 3454       | 456     | **87%** |
| **平均**                  | **1214**   | **294** | **65%** |

10 个不同复杂度任务的真实 API 数据，平均节省**65%**的输出 Token。而且 Caveman 只影响输出 Token，不影响推理/思考 Token——**该想的还是那么深，只是说出来的时候更短了**。

### 3. 不止是“说话”变短：记忆文件也能压缩

Caveman 还有一个隐藏技能——`/caveman-compress`命令可以把你的`CLAUDE.md`、项目笔记等记忆文件也压缩成原始人风格。

实测数据：5 个真实记忆文件平均从 898 个 token 压缩到 481 个 token，节省**46%**的输入 Token。代码、URL、路径等关键信息**字节级保留**，一个字都不丢。

**这意味着你每次开启新会话，加载的上下文就已经小了将近一半。**

### 4. 实时统计：省了多少 Token，一目了然

`/caveman-stats`命令会显示当前会话的 Token 节省量和累计节省总额。Claude Code 的状态栏还会直接显示一个徽章：`[CAVEMAN] ⛏ 12.4k`——告诉你这一辈子已经省了多少 Token。

### 5. 30+工具全兼容，一键安装

Caveman 支持 Claude Code、Codex、Gemini CLI、Cursor、Windsurf、Cline、GitHub Copilot 等**30 多种 AI 编程工具**。

安装只需要一行命令：

```bash
# macOS / Linux / WSL
curl -fsSL https://raw.githubusercontent.com/JuliusBrussee/caveman/main/install.sh | bash

# Windows (PowerShell)
irm https://raw.githubusercontent.com/JuliusBrussee/caveman/main/install.ps1 | iex
```

~30 秒完成，自动检测你已安装的 Agent 并配置。触发方式也很灵活：输入`/caveman`或直接说“talk like caveman”。想恢复正常模式就说“normal mode”。

### 6. OpenClaw（龙虾）原生支持

如果你在用 OpenClaw（就是那个 Star 数 37 万的“龙虾”），Caveman 可以直接集成。安装时加上`--only openclaw`参数，Caveman 会自动在 OpenClaw 的工作目录里放下技能文件和 SOUL.md 标记。**Lobster mouth now small. Brain still big.**

## 03 ｜横向对比

Caveman、RTK 和 9Router 都是帮你在 AI 编程里省钱的工具，但省钱的方式完全不同：

| 维度         | **Caveman**                 | **RTK**                             | **9Router**                       |
| :----------- | :-------------------------- | :---------------------------------- | :-------------------------------- |
| **省钱方式** | 让 AI**少说话**（压缩输出） | 让 AI**少看废话**（压缩命令输出）   | 让 AI**用便宜的渠道**（智能路由） |
| **作用对象** | LLM 的输出 Token            | 命令执行的输出                      | 请求路由和订阅管理                |
| **压缩对象** | AI 生成的回答文本           | `git diff`、`npm install`等命令输出 | 多账号/多提供商轮询               |
| **节省幅度** | **65%**输出 Token           | 60-90%命令输出                      | 取决于订阅组合                    |
| **使用方式** | 一句话激活，无感运行        | 自动 Hook，无感运行                 | 配置路由策略                      |
| **兼容工具** | 30+ AI 编程工具             | Claude Code/Cursor 等 12 款         | 所有 AI 编程工具                  |

**Caveman 和 RTK 可以叠加使用**——RTK 帮你压缩命令输出（输入 Token），Caveman 帮你压缩 AI 回答（输出 Token）。两把刀一起砍，账单直接对半折。

## 04 ｜典型场景和避坑

**适合谁用？**

- **深度使用 Claude Code、Codex 等按 Token 计费工具的重度用户**：65%的输出 Token 节省，意味着同样的预算能多跑将近三倍的对话
- **觉得 AI 回答太啰嗦、只想看技术干货的开发者**：Caveman 的输出风格干净利落，直击要害
- **在用 OpenClaw（龙虾）的“养虾人”**：Caveman 原生支持 OpenClaw，让龙虾说话更精炼
- **需要处理大量短问答的场景**：每次回答省一点，积少成多

**不推荐给谁？**

- **需要 AI 输出完整文档、教程、解释性内容的场景**：Caveman 的输出是片段式的，不适合生成正式文档
- **对回答语气有要求的场景**：Caveman 的“原始人”风格可能会显得过于生硬
- **完全不用 AI 编程工具、只做纯对话的场景**：Caveman 是为编码场景设计的，纯聊天用不上

## 05 ｜快速上手指南（2 分钟）

**第 1 步：一键安装**

```bash
curl -fsSL https://raw.githubusercontent.com/JuliusBrussee/caveman/main/install.sh | bash
```

脚本会自动检测你已安装的 AI 编程工具并完成配置。

**第 2 步：激活 Caveman**
在 Claude Code 或你的 AI 工具中输入：

```
/caveman
```

或者直接说“talk like caveman”。

**第 3 步：切换压缩级别（可选）**

```bash
/caveman lite      # 轻度精简
/caveman full      # 标准原始人（默认）
/caveman ultra     # 极致压缩
/caveman wenyan    # 文言文模式
```

**第 4 步：查看省了多少 Token**

```bash
/caveman-stats
```

会显示当前会话的 Token 节省量和累计总额。

想恢复正常模式，直接说“normal mode”即可。

## 06 ｜总结

Caveman 解决了一个被很多人忽略但真实存在的问题：**AI 太有礼貌了。**

每一个“I'd be happy to help”、每一个“Let me explain”、每一段重复的铺垫——都是你按 Token 付的钱。在 Claude Code 这种按量计费的工具里，礼貌是有价格的。

Caveman 的解法很简单：**把 AI 的“嘴”变小，但脑子不变。**

砍掉客套话、删掉重复、保留干货——同一段技术解释，从 1180 个 token 压缩到 159 个。节省了 87%，信息量一分没少。

GitHub 81.4K Star，OpenAI 员工都在用。这不是一个“玩具”，这是一个被大量开发者验证过的、每天都在帮你省钱的工具。

**Why use many token when few token do trick**。
