## 01 ｜核心逻辑

Cursor、Copilot 是你的“副驾驶”。你给指令，它帮补代码、改文件——**最终还是你在开车**。

OpenHands 不一样。你给它一个任务描述，它自己拆解步骤、规划路径、写代码、建文件、开终端跑命令、装依赖、打开浏览器查文档、跑测试发现报错、自己修，最后给你提一个 PR。整个过程你可以盯着看，也可以走开喝杯咖啡回来检查结果。

**OpenHands 的本质，不是更聪明的自动补全，而是一个能独立完成软件工程任务的自主编程代理。**

支撑这个能力的，是三样东西的组合：

- **Docker 沙箱隔离**：Agent 在本地跑（密钥不出机器），但所有代码执行都在 Docker 容器里。哪怕 Agent 犯了错误跑了`rm -rf /`，也只影响容器内部，你的主机安全得很。
- **CodeAct 架构**：它不依赖预置工具列表。遇到需要搜索代码库的情况，Agent 自己写一个`grep`脚本；需要重构模块，自己写一个 transform 脚本。任何能用 Python 表达的任意操作，都可以执行。
- **模型自由选择**：支持 Claude、GPT、Gemini、本地模型……你在 OpenHands 里用 Claude Sonnet 4，价格就是直连原价，中间没有平台加成。不想付钱也可以接入免费 Minimax 模型直接开始。

## 04 ｜典型场景和避坑

**适合谁用？**

- **想自动化处理代码问题、修 Issue、做重构的开发者/团队**：开一个 Issue，OpenHands 自己理解代码、跑测试验证、推上 PR。
- **想集成 AI 编程能力到 CI/CD 流程的 DevOps 工程师**：SDK 和 CLI 模式完美适配自动化流水线。
- **想尝试自主 AI 程序员但不想付费的个人开发者**：完全 MIT 开源，自己部署，模型可以选择本地 Ollama 或任意低成本的云端 API。
- **需要透明、可审计的 AI 工程化方案的企业用户**：代码全开源、沙箱隔离、日志清晰。

**不推荐给谁？**

- **只需要编辑器内智能补全的用户**：它的优势在于自主执行，不是代码补全。
- **只想跑单一简单对话的场景**：直接调用模型 API 比部署 Docker 环境更轻量，比如 RAG 问答就无需给它动手改造工程的能力。
- **对服务器资源敏感、不想跑 Docker 的用户**：虽然也可以通过 CLI 模式运行，但核心特性依赖 Docker 沙箱，轻量化依赖项目最新的成熟度还需要时间验证。

## 05 ｜快速上手指南

最推荐的部署方式是 Docker：

**1. 本地 Docker 运行（最推荐）**

```bash
docker run -it -p 3000:3000 \
  -e WORKSPACE_MOUNT_PATH=$HOME/code \
  -v /var/run/docker.sock:/var/run/docker.sock \
  openhands/openhands:0.25.0
```

浏览器打开`http://localhost:3000`，从 GUI 中选择模型提供商并填入 API Key 即可。

**2. CLI 模式运行**

```bash
# 从GitHub克隆并安装
git clone https://github.com/OpenHands/OpenHands.git
cd OpenHands
python -m openhands.core.cli
```

**Windows 用户特别提醒**：用 Docker 跑最简单。Windows 上必须先安装 WSL2 和 Docker Desktop，官方文档中 Windows 原生命令行部分的支持成熟度不如 macOS/Linux，所以强烈建议走 Docker Desktop 路线，省去一箩筐环境兼容的零碎琐事。

**3. OpenHands Cloud（免费试用）**

访问`app.all-hands.dev`，用 GitHub/GitLab 账号登录，选择免费的 Minimax 模型，在浏览器中直接开始使用，零部署成本。

**验证安装成功**：在 GUI 或 CLI 中输入`/help`，查看可用指令列表。用简单的“在这个目录里建一个`README.md`”先试试水，确保任务能被正确执行。
