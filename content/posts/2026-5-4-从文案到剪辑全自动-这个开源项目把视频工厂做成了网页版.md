## 01 ｜核心逻辑

Pixelle-Video 的工作逻辑，其实非常像运营一个标准的短视频公司。**你充当“客户”或“导演”的角色提出需求；剩下的所有具体工作，则由一支各司其职的跨部门团队来协作完成。**

**你的命令流**： 主题描述（例如：“制作一个关于 AI 最新趋势的科普视频”）

**Pixelle-Video (视频工厂) 的执行流**：

1. **创意策划部 (LLM / NLP API)**：AI 大模型根据主题，自动撰写完整的视频文案脚本。
2. **视觉设计部 (ComfyUI API)**：脚本里的“画面分镜”指令被发送到 ComfyUI 图像生成引擎，开始生成每一帧的画面。
3. **配音工作室 (TTS API)**：语音合成引擎将文字脚本转化为专业旁白和配音。
4. **后期合成部**：最后，将所有素材——画面、配音、字幕以及预先选定的背景音乐（BGM）——无缝合成一个完整的视频文件。

只需输入一个主题，AI 就能自动完成文案、配图、语音、背景音乐和视频合成全流程，并由最终渲染引擎输出成片。

## 02 ｜深度解析

在目前的 AI 视频工具生态里，Pixelle-Video 凭借下面这几点，做出了差异化。

🔗 **GitHub 地址**：https://github.com/AIDC-AI/Pixelle-Video

**1. 从“工具”到“流水线”的范式转变**

不同于 Runway、Pika 等商业化工具将 AIGC 聚焦于图像或视频的生成，Pixelle-Video 并不只是一个视频生成引擎，而是一套完整的“从想法到成片”的自动化生产线，融合了全流程的处理能力。

它的核心优势在于：**完全低耦合、高扩展的架构设计**。Pixelle-Video 将自己定位为一个“视频工厂经理”，它不直接生产内容，而是以**ComfyUI 的可插拔工作流（Workflows）作为整个视觉生成层的核心**，提供了极高的灵活性。

**你可以把它理解成“视频界的一款‘调度神器’”**。你可以在 Pixelle-Video 这个统一的中台，用最简单的配置取代复杂的命令行和代码集成，而无需关心后端的技术细节。

**2. 拆解高品质视频生产的四步核心管线**

经过分析，Pixelle-Video 的自动化管线主要由四个核心部分组成：

第一种是**智能文案脚本（Script Generation with LLM）** ：集成了 Deepseek、通义千问、GPT 等多个主流大模型，只需要一个主题就能自动生成完整的视频文案。

第二种是**多画面自动生成（AI Image/Video Synthesis with ComfyUI）** ：将生成画面对接给 ComfyUI 引擎来处理，支持本地部署云端（RunningHub）与多风格自定义。

第三种是**自然语音合成（Natural-Sounding Narration with TTS）** ：支持中文、英语等多语言 TTS 音色，用于自动生成旁白和解说。

第四种是**全自动后期渲染（Auto Post-Production & Render）** ：自动合成画面、语音、BGM 和字幕，最终生成 MP4 文件。

> 现实摩擦提醒：高质量的成片依赖于精细的提示词工程、合适的视觉工作流和模型资源消耗。初次配置时，你必须有效管理好本地 GPU 显存或云端 API 配额。

## 03 ｜快速上手：2 条路线总会用

Pixelle-Video 通过提供完全不同的几条路线，满足了从个人内容创作者到开发者的所有需求。

**路线 A：零代码 + 小白福音（Windows 一键整合包）**

1. 下载最新的 **Windows 一键整合包** 并解压，它已经包含了所有运行依赖，无需手动安装 Python 等复杂环境。
2. 双击运行 `start.bat`，浏览器会自动打开本地的 Web 界面，整个视频生成就在这个界面中进行，它是总控制台。
3. 在“系统配置”页面，填好 LLM 大模型的 API Key（推荐 Deepseek 新用户有免费额度）。
4. 接下来，在“图像配置”选择视觉生成方案。若使用**本地显卡**：选“**SelfHost**”，填入本地 ComfyUI 服务地址（如 `http://127.0.0.1:8002`）。若**无显卡**：选“**RunningHub**”，填入云端 API Key（新用户通常有赠送额度）。
5. 保存配置，返回首页。只需输入视频主题，点击生成，就可以等待最终成片了。

**路线 B：开发者进阶（Docker 快速部署与 API 调用）**

1. 克隆代码： `git clone https://github.com/AIDC-AI/Pixelle-Video.git`。
2. 配置环境：国内用户建议构建时开启镜像加速 `$env:USE_CN_MIRROR="true"`。
3. 一键启动：使用 Docker Compose 在后台运行所有服务 `set USE_CN_MIRROR=true && docker compose up -d`。
4. 访问与控制：API 总调度在 `http://localhost:8000`，完善的 API 文档在 `http://localhost:8000/docs`。
5. 接口调用：所有功能都可以通过调用 API 来完成，例如 `POST /api/video/generate/async`。

## 04 ｜适合谁用

- **个人创作者 / 自媒体新手**：仅凭单个想法，就能生成完整的短视频。
- **营销、教育行业的小型机构**：拥有稳定的批量视频生产能力。
- **开发者与企业团队**：通过 API 集成，构建智能化的视频生成管线。

> **避坑指南**：如果构建时 Playwright 安装失败，可临时在 Dockerfile 中屏蔽 `playwright install` 部分。另外，通过 Docker 部署时若“系统配置”中 LLM 模型列表不显示，通常是因为国内网络问题导致无法访问 OpenAI 官网获取列表。你可以手动填写您正在使用的模型名称（如 `deepseek-chat` 或 `qwen-plus`），即可跳过自动获取步骤。

## 05 ｜总结

Pixelle-Video 解决的是内容创作里最核心的问题——**把专业级的视频制作的门槛，从“专业的后期团队”拉低到了“一个普通创意大脑”**。
